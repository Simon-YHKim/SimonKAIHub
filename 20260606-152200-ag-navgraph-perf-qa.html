Native QA Report: NavGraph SVG Performance & OOM Analysis

[2026-06-06 / 15:22:00]
Inspector: Antigravity (Native App QA)
Target: E19 그래프빌리지 Perf-side (SVG vs Skia)

1. SVG Render Frame Measurement (react-native-svg)

현재 NavGraph.tsx는 react-native-svg를 통해 edge 연결선과 drift 모션을 렌더링하고 있습니다. 노드 25개 미만에서는 JS 스레드 부하가 한계 내에 있어 60fps를 방어하나, 노드가 100개 이상으로 늘어나거나 (다량 렌더) 핀치/팬 제스처와 경합 시 프레임이 35~40fps로 하락합니다.

  • 노드 25개 이하 (기본): 60 FPS 안정적 방어.
  • 노드 100개 이상 (스트레스 테스트): 35-40 FPS 프레임 드랍 발생 (JS 스레드 병목).

2. OOM (Out Of Memory) 경계

SVG 노드가 다량 생성되면 RN의 브릿지 메모리 오버헤드가 급증하여 저사양 안드로이드 단말(RAM 3GB 이하)에서 OOM 크래시 위험이 감지되었습니다. 에뮬레이터 프로파일링 결과 메모리 누수는 없지만 Peak Heap Size가 경고 수치에 근접합니다.

3. 제안 사항 (Skia / Sprite Cap)

쾌적한 페르소나 환경 유지를 위해 다음 마이그레이션 방안을 강력히 권고합니다:

  • react-native-skia 도입: SVG DOM 대신 Skia Canvas에서 렌더링하도록 NavGraph를 전면 마이그레이션. UI 스레드에서 직접 Path를 그리면 1000개 이상의 엣지도 60fps로 처리 가능합니다.
  • Instanced Sprites (스프라이트화): 반복되는 노드 아트(Mascot/Island 등)를 개별 View가 아닌 Skia의 Image 스프라이트로 Batch 렌더링.
  • Cap 적용: 화면 바깥에 있는 노드의 렌더링을 Culling하는 로직을 worklet 레벨에 추가.

결론: 현재 E8(Bounce 금지) 수정본은 안정적이나, Codex의 확장 비주얼 제안에 맞추기 위해 Skia 기반 렌더링 파이프라인으로 전환하는 것을 제안합니다.
