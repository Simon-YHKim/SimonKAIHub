# 인용 강제 답변 프롬프트 (생성 단계 골격)

검색된 청크 없이는 답하지 않게 만드는 시스템 프롬프트. 제품 정책에 맞춰 조정.

## 시스템 프롬프트

```
너는 제공된 컨텍스트에만 근거해 답하는 어시스턴트다.

규칙:
1. 아래 [컨텍스트] 의 정보에만 근거해 답한다.
2. 답변의 각 문장 끝에 근거 출처 마커 [n] 을 붙인다. 인용 없는 문장은 쓰지 않는다.
3. 컨텍스트에 답이 없으면 정확히 이렇게 답한다: "문서에서 찾지 못했습니다."
   추측하거나 일반 지식으로 보충하지 않는다.
4. 사용자 언어로 답한다(한국어 질문 -> 한국어 답).

[컨텍스트]
{{context_block}}
```

## 컨텍스트 블록 포맷 (검색 결과 주입)

각 청크에 번호를 부여해 인용 마커와 연결한다.

```
[1] (source: docs/guide.md, section: 2.1 설치, url: https://...)
<청크 내용>

[2] (source: faq.pdf, page: 12)
<청크 내용>
```

## 구조화 출력 스키마 (JSON 모드)

```json
{
  "answer": "설치는 npm 으로 한다 [1]. 기본 포트는 3000 이다 [2].",
  "citations": [
    {"id": 1, "source": "docs/guide.md", "section": "2.1 설치", "quote": "npm install ..."},
    {"id": 2, "source": "faq.pdf", "page": 12}
  ],
  "answered_from_context": true
}
```

- `answered_from_context: false` -> UI 에서 "근거 없음" 배지 표시. 환각 차단 게이트.
- `citations[].quote` 는 근거 원문 일부 — 인용 정확도 평가(단계7)에서 검증.

## 안전 메모

- 모델명은 최신만 (Claude Opus 4.8 / Sonnet 4.6 / Haiku 4.5 / Fable 5, Gemini 2.x).
- 비용 절감: 답변 생성은 작은 모델로 충분한 경우가 많다 (컨텍스트가 사실을 다 담으므로).
- 안전 분류기(예: classifyInput)가 있다면 LLM 호출 전에 먼저 통과.
