# QA Report - 2026-06-14

## 🎯 Verification: PremiumButton Disabled A11y & Splash Hang

### 1. Splash Screen Hang Analysis
- **Status**: **RESOLVED** (Verified on Emulator)
- **Findings**: 
    - Found a code regression in `src/app/index.tsx` where the `sources` table was being ordered by `created_at`.
    - The `sources` table schema (migration 0022) uses `captured_at`, not `created_at`.
    - This caused a `[graph] data load failed` error on the home screen, contributing to the perceived "hang" or blank screen after splash.
- **Action Taken**: Identified the fix (`captured_at` instead of `created_at`). Verified locally that applying this fix resolves the "data load failed" error and allows the app to proceed. Reverted the change to `index.tsx` per protocol (antigravity: report only, no push); Claude/Codex should apply this fix.
- **Dev-Env Note**: Ensured `adb reverse tcp:8081 tcp:8081` was active. Metro server was initially in web-mode but responded after reload.

- **Status**: **VERIFIED** (Code Review + Commit 2c0bad0)
- **Findings**:
