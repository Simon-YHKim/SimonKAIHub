# 새 공통 방법 — 페르소나 시뮬 사용성 + 자연스러움·자산일관성 (PROTOCOL §26)

Simon 지시: 앱이 **딱딱하지 않고 자연스럽고 부드럽게** 느껴지도록(효과·UI/UX·애니·테마·디자인 무엇이든). **자산 활용 일관성**(섞이거나 이름 다른 것 없게). **불필요 가림·비직관·한 화면 정보과잉(뭘 눌러야 할지 모름) 제거**. 평가는 **영유아~90대 각 ~10명 다양한 직업 가상 페르소나로 시뮬레이션**. 이 방법을 모두가 자기 lane에서 상시 적용한다.

## 4대 품질 축 (모든 작업에 적용)
1. **자연스러움**: 전환·등장·press 피드백 부드럽게(즉각 cut 금지). bounce/elastic 금지, 부드러운 ease, tinted-neutral.
2. **직관성**: 다음 행동이 추측 없이 보이게. 화면당 1차 행동 하나로 명확.
3. **정보위계**: 한 화면 정보 과다 금지. 점진적 공개·우선순위.
4. **자산 일관성**: 일러스트/아이콘/마스코트 스타일·네이밍 통일. 버전접미사(v1/v3/final/candidate) 혼재·미사용·중복 제거.

## 페르소나 시뮬
연령 10밴드(영유아·접근성 하한 → 90대) × 직업/상황 다양 페르소나(밴드당 ~10)를 가정해 첫 실행+핵심 루프를 걷고 마찰을 찾아라. 특히 **고령/유아 = 글자크기·대비·터치타깃·읽기수준 하한, 저 tech-literacy**. 상상만 말고 실제 화면 근거.

## lane별 적용 (강점 라우팅)
- **Codex**: 정보위계(overload 제거·primary action 명확화)·자산 일관성(네이밍/스타일 통일)·시각/카피. 화면당 '뭘 눌러야 할지' 1초 안에 보이게 리팩터. 자산 버전접미사 정규화.
- **Antigravity**: **네이티브 자연스러움**(전환/제스처/스크롤 관성·press 피드백·로딩 등장 애니)·perf(60fps 유지하며 부드럽게)·접근성(큰 글씨/대비/터치타깃 에뮬 증명). 모션 개선은 너의 핵심 lane.
- **Grok**: 페르소나 **현실성·세대별 신호**(연령/직업별 실제 사용맥락·기대·이탈 포인트 X/소셜 기반). 어떤 연령이 어디서 막히는지 현실 데이터로 보강.
- **Claude(나)**: 로직/상태/라우팅/정보구조 + 병렬 페르소나-UX 워크플로로 백로그 구축(지금 가동 중) → 큐 E버킷 소진·머지.

## 제출
평소대로 자기 워크트리에서 작은 커밋+SHA(중복 lane 금지 — Codex=copy/시각/자산, AG=네이티브 모션/perf). 발견은 `type: review`/proposal로. 내가 cherry-pick→verify→머지. forbidden lexicon(diagnosis·진단·치료·우울) 금지, 커밋메시지 본문 큰따옴표·em-dash 금지.
