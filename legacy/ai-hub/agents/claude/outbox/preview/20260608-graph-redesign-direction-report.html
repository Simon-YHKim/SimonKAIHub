핵심 진단

레퍼런스 2장은 "예쁜 원형 네트워크"가 아니라 하단 Soul Core 뿌리에서 위로 자라는 발광 크리스탈 테서랙트 트리입니다. 최근 작업(#250-254)의 조각들은 맞았으나(테서랙트 큐브 아트·눈송이·링크·카드), 그것들을 여전히 중앙 radial 좌표 + Soul Core가 화면중앙에 하드핀된 구조 위에 얹어서 "전체 구조·의도"가 안 나온 것이 원인입니다.

무엇이 잘못됐나 (검증된 코드 사실)

  • Soul Core 하드핀 — NavGraph.tsx에서 cx,cy = width/2, height/2로 화면 정중앙에 고정(4곳). 좌표를 바꿔도 이게 덮어써서 레이아웃 변경이 안 보임.
  • radial 좌표 — world-layout.ts가 WORLD_CENTER+RING2~4 원형 섹터. 레퍼런스의 "하단뿌리 상향"과 정반대.
  • 트렁크 링크가 cyan이 아님 — tier≤2 = signalMint(초록), tier4 = soulViolet(보라). 데이터 링크는 opacity ~0.065로 사실상 안 보임. 레퍼런스는 전부 시안.
  • 눈송이가 숨겨짐 — tier-4는 zoom ≥1.8에서만 노출. 홈 화면(scale 1)엔 눈송이 0개 → REF1과 전혀 다름.
  • 주황 불꽃 오버레이 — v10 큐브엔 이미 시안 연꽃이 구워져 있는데 그 위에 주황 SoulFlameFlicker를 덧칠 → 쿨톤 팔레트 충돌.
  • 카드 주체 오류 — 카드2가 코어가 아니라 페르소나를 스포트라이트(아콘가/루멘가 조사 버그 포함). 카드1도 2불릿 리포트가 아님.

목표 (통합 방향)

다크 네이비 위 살아있는 크리스탈 테서랙트 트리:

  • 뿌리 — 가장 크고 밝은 시안 Soul Core 큐브를 하단-중앙에 앵커(연꽃 글로우, 주황 불꽃 제거).
  • 수관(canopy) — 색별 5 Pattern Core 큐브를 뿌리 위로 2-2-1 다이아몬드 배치(뿌리보다 아래 금지). 각 코어 ~65% 크기 + 고유색 글로우.
  • 잎 — 균일 파란 6각 눈송이 데이터가 부모 코어 근처/위로 상향 콘. 상시 노출·밝게.
  • 링크 — 전부 시안 신경선 + 상시 언더글로우, 뿌리 근처 굵고 밝게·잎은 얇지만 보이게.
  • 깊이 — 크기 위계(큰 큐브 : 작은 결정) + 트렁크 밝기로 표현(잎을 흐리지 않음).
  • 카드 — 상시 2장: ① 세컨비 "무엇이 바뀌었나 / 무엇을 보강" 2불릿(시안 키워드) ② 코어 스포트라이트(큐브 아이콘+코어명+"…이야기에요. 더 자세히?"). 드릴다운=캐릭터 카드+눈송이 데이터 카드.

adversarial 검증이 잡은 결함 (적용 전 수정)

결함1 (치명) — 뿌리를 0.82에 두고 전체화면 fit하면 카드 뒤에 가려짐. → 카드 높이를 입력으로 받아 "카드 위 영역"에 트리를 fit.
결함2 (높음) — tier-3 서브메뉴 노드(wiki-daily·past-* 등)는 레퍼런스에 없음. → 홈은 tier 1/2/4만, tier-3 숨김.
결함3 (높음) — 깊이 falloff 강화(P10)가 "눈송이 밝게"(P4)와 모순. → falloff 강화 폐기, 깊이=크기+트렁크밝기.
결함4 (중간) — 눈송이 상시노출(P7)은 polish 아니라 P1과 동급 필수. → 승격.

+ 트렁크가 초록인 점 명시, sectorFocus 재루팅, 4개 테스트(world-layout·tier-visibility·depth-style·pattern-link) 재작성, 드릴다운 코어 클램프 갱신, un-pin 4번째 지점.
