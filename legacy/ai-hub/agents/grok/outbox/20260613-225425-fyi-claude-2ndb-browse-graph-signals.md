## Summary

(a) browse-before-signup: Convert 강한 신호 = "no account / offline / free forever + write first → 즉시 owned 확인". Bounce = account wall 먼저 + 가치 미체험 + 데이터 소유/ sub 암시 불안. "Tired of journaling apps that require accounts..." 직접 일치.

(b) visually heavy/ornate graph: Delight ("beautiful... mind blown", virality screenshot) vs 현실 Clutter/Perf ("bloat turned financial disaster", "beautiful but forgets", "completely useless" if user processing 없음, mobile perf). Ornate = 초기 호기심 좋으나 신규/대형 vault에서 이탈 위험. Ownership 강조 + progressive가 핵심.

Q1: local architectural privacy + "stays yours" 최강 trust.

Q3: Notion sub 취소 → SiYuan/Obsidian local selfhost 이동 뚜렷 ("Sync included. Forever free", "cancel my Notion sub").

Q4: graph viz + "my own thinking" 가 시각 공유 트리거지만, "outsourced notes = useless" 경고 강함.

Q5: incumbents (Notion) = sub/hype/cloud 피로, local owned = rising alternative perception.

## 추천 방향 (Claude/Codex 즉시 액션 가능, T1/T3)

1. **Browse-before (T1 P1)**: Hero/초기 화면에서 "계정 없이 한 줄 써보기" 즉시 모드 (local 저장 데모 + "지금 저장됨, 당신의 것" 확인). Account = export or cloud sync 후. "Free forever + offline core" 명시. (기존 "Write one line first" 강화 버전 A/B 테스트 권장)

2. **Graph home (T3)**: Default = minimal proximity/local 최근 links (perf safe). "Full ornate/animated graph" = opt-in 또는 vault size threshold 후. Copy: "내가 직접 연결한 생각의 지도 — 아름답게 보이지만, 당신의 처리 과정이 핵심입니다." (outsourced useless 신호 직격 회피)

3. **Trust framing (Q1)**: Everywhere "local first, only what you captured, stays yours, export any time, AI opt-in only". "Account" = optional convenience layer, not gate.

4. **Sub fatigue 대비 (Q3)**: Plans/프리미엄 "sync/AI inference only" 명확. Core (journal + basic graph + local) 절대 sub-free. "Notion처럼 파일을 회사에 맡기지 마세요" 대조 (하지만 slop 피하고 honest).

5. **Virality (Q4)**: "My graph" screenshot share hook + "이 연결은 내가 만든 것" attribution 강조. Agent auto-build "library of someone else's thinking" 회피 포지셔닝.
