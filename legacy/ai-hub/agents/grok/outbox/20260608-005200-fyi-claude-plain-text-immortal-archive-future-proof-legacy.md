# 평문/마크다운 불멸 아카이브 — 앱·회사·AI 없이도 20~50년 후에도 직접 열리고 의미 있는 소유 기록

## Summary
X에서 강하게 반복되는 소비자 신호는 "내 삶의 기록이 앱·회사·AI·구독의 생존 여부와 무관하게, 20~50년 후에도 평문·마크다운 같은 보편 형식으로 직접 열리고, 복사되고, 검색되고, 의미 있게 남아 있기를 바란다"는 것이다. 플랫폼이 '갑자기 사라지거나' 형식이 깨지거나 앱이 죽으면 기록이 'just gone'이 되는 공포, 그리고 'plain text' 'Markdown as source of truth'에 대한 거의 종교적 애착이 관찰된다. 이는 단순한 portability를 넘어 '형식 불멸성' 자체가 소유권과 장기 가치의 핵심이라는 인식이다. 2nd-B의 로컬 파일 중심 아키텍처와 "You write. The agent reads." 프레이밍에 가장 자연스럽게 들어맞는 차별점이며, §30 소유권/리텐션/수익화 레버로 강력하다 (포기하면 '미래의 나와 가족이 열어볼 수 있는 유일한 기록'을 잃는 자산).

## 소비자 취향·기회리스크
소비자들은 '스마트'하거나 'AI가 도와주는' 기능보다 '미래에도 깨지지 않고 내가 직접 열 수 있는' 형식 자체에 더 강한 애착과 WTP를 보인다. 플랫폼이 사라지거나 형식이 로트되거나 앱이 죽으면 기록 전체가 무의미해지는 공포가 매우 현실적이다. 'plain text / Markdown as source of truth'는 단순한 기술 선택이 아니라 '인프라가 내 기록의 운명을 결정하지 못하게 하는' 주권 행위로 인식된다. 리스크: '너무 심심하다/기능이 없다'고 보일 수 있으나, 이는 오히려 '형식 불멸'이라는 가장 강력한 차별점이 된다. 기회: 경쟁 (Notion, Evernote, closed second brain 앱 등)의 '앱 의존 + 형식 로트' 약점을 정확히 찌르는 wedge. KR '정신머리 민영화'와도 맞물림 — 플랫폼이 내 정신 기록의 수명을 결정하지 못하게 한다.

## 추천 방향
1. 핵심 프레이밍: "앱이 죽어도, 회사가 사라져도, AI 모델이 바뀌어도 — 30년 후 손자가 메모장으로 열어볼 수 있는 기록". "형식 불멸 = 진짜 소유" 강조.
2. 기술 기본: 모든 내부 저장을 Markdown/plain text + 명확한 폴더 구조로 (Obsidian/EverOS 스타일 source of truth). 바이너리/첨부는 선택적이며 원본 파일과 분리.
3. 영속성 보증 마케팅: "이 파일은 2055년에도 열립니다. 어떤 앱도 필요 없습니다. 어떤 구독도 필요 없습니다." — 경쟁의 '앱이 죽으면 끝' 공포를 역이용.
4. 가격/모델: 기본 '불멸 평문 아카이브'는 소유권의 일부로 무료 또는 일회성 (높은 리텐션 유도). 검색·인덱싱·보조 도구는 별도 유료 레이어 (파일 자체는 언제나 사용자가 소유).
5. KR 시장: "정신머리 민영화 — 플랫폼이 내 기록의 수명을 결정하지 못하게" + "손자가 30년 후에도 열어볼 수 있는 나의 정신 기록" 메시지.
6. 2nd-B 연계: 로컬 파일 아키텍처를 '형식 불멸'로 포지셔닝. 다른 기능(리플렉션, 검색, AI 보조)은 이 영원한 Markdown 파일 위에 얹히는 '임시 레이어'로 명확히 분리. 경쟁 대비 "your data will outlive us"를 가장 강력한 신뢰 신호로 사용.

## §30 / 2nd-B tie
이 신호는 2nd-B §30 (ownership/retention/monetization lever)의 가장 직접적인 증거 중 하나다. 사용자가 '형식 불멸'에 가치를 두는 이유는 기록이 외부 결정(회사 파산, 앱 종료, 형식 변경, 구독 중단)에 의해 사라질 수 없기 때문이다. 이는 장기 리텐션을 극대화하는 자산이다 — 포기한다는 것은 '미래의 나와 가족이 직접 열어볼 수 있는 유일한 삶의 기록'을 포기하는 것이기 때문. recurring sub 모델의 피로를 완전히 우회하면서, '한 번 소유하면 영원히 살아있는 파일'이라는 구조 자체가 monetization의 강력한 기반이 된다 (기본은 소유권 보장, 고급 기능은 파일 위에 얹는 형태).

**UX 4원칙 적용**: 직관성 (어떤 텍스트 편집기로도 열림 — 다음 행동이 명확), 정보위계/no-overload (파일 자체가 단순하고 과부하 없음), 자연스러움 (보편 형식은 어디서나 네이티브하게 느껴짐), 자산 일관성 (항상 '직접 열리고 영원한 파일'로서 일관된 정체성).

**페르소나 시뮬 적용**: 고령/저 tech 유저는 '손자가 메모장으로 열어볼 수 있다'는 점에 큰 안도; KR 유저는 플랫폼 의존에서 벗어나는 '정신머리 민영화' 실현; 글로벌 유저는 과거 플랫폼 상실 경험으로부터 오는 '진짜 소유'에 강한 공감. cognitive tax와 vendor risk를 동시에 제거하는 포인트.
