## §35 Self-Panel (5 렌즈 · 적대적 검증 포함 · 제안자=grok, 심판=신호 기반)

### 렌즈 1: KR 20-30대 지식노동자 / 일상 성찰 사용자
- X 신호: "투다" (social+diary, self book for friends, stickers, 댓글) — 가벼운 감정 기록 + 자기 객관화. "마성의 토닥토닥" 앱 추천 사례 (감정 기록·객관화 도움).
- "일기 앱" 언급 시 privacy/sync (local-first vs cloud) 우려 빈번. 임상적·무거운 프레임보다 "친구와 나누는 책" 같은 일상 framing 선호.
- Advisory: 첫 기록을 "임상 일지"가 아닌 "오늘의 한 조각"으로. KR 사용자에게 "부담 없이" + "내 것" 느낌 강조. 스티커·경쾌 요소는 별도, 핵심은 진입 장벽 0.

### 렌즈 2: 글로벌 PKM / Second Brain 사용자
- X 신호: "Files.md plain-MD local-first second brain" — "Everything in Markdown files, offline-ready... One big idea per note... No build steps. No plugin churn. Just open index.html." "fits in your head", "your thoughts don’t get buried in UI layers".
- "I wish ... Truly a second brain similar to Obsidian" (Greg Isenberg wishlist) vs 실제 "when it comes to actual writing, I still prefer using my 'first brain' with just a simple, blank document" (Obsidian user).
- Advisory: second brain 포지셔닝 좋으나 T1 진입은 "blank document" 단순함을 최대한 살려라. UI 레이어 최소화가 첫 write comfort의 핵심.

### 렌즈 3: 영속성·장기 소유 중시 사용자 (local ownership, no lock-in)
- X 신호: Harbour Journal (privacy-first local AI journaling): "The AI is not a chatbot bolt on... processing everything on your Mac, no exceptions. There is no cloud, no subscription, and no compromise. You can even import your entire Day One journal." 
- Pen & paper 선호 근거: "Physically writing them stores & encodes... You can review it at any time (external storage) And it will likely be remembered/acted on (encoded in brain)" (DAN KOE).
- "The saving feature is awesome. Save only what you choose to keep is how people naturally work."
- Advisory: "모든 기록은 네 로컬 파일. 클라우드 없이. 언제든 export·이동·읽기. 네 것이 영원히." permanence를 "local files + plain format"으로 실체화. Day One import 같은 migration path도 신뢰 신호.

### 렌즈 4: 완벽주의 / 부담 회피 / "해야만 한다" 불안 사용자 (low-pressure entry)
- 강력 신호 (반복 재현): "Why do you save so many things? ... Because saving feels like having. And having feels like being. ... The save button is an emotional pacifier. It soothes the anxiety of 'I should be doing more' without requiring you to actually do more. ... A graveyard of good intentions. A museum of the self you keep promising to become." (Kpaxs, high engagement).
- "right-click save is a form of Stockholm syndrome... you buy digital goods you don’t own!"
- Advisory: 첫 write/save 플로우를 "수집=성취"가 아닌 "한 조각 놓기(실제 행동)"로 framing. Aspirational 모으기 강조 금지 — 오히려 불안·이탈 유발. "한 줄부터. 판단 없이." 가 핵심 진입 문구.

### 렌즈 5: 습관 형성 · retention (cross-cutting)
- "Products stick when they stop feeling like tools and start becoming part of the workday. ... retention lives where the habit forms."
- "A journaling app is one of those projects that sounds simple but gets deep when you think about privacy and sync."
- Advisory: T1에서 "tool" 느낌 최소화. Workday/일상 일부가 되는 순간 (첫 기록 후 즉시 가벼운 피드백 + 로컬 저장 확인) 이 retention의 관문.

## 종합 X/KR 신호 요약 (2-pass 재현성 확인)
- Local-first + plain format + no-cloud = strongest "영속 소유감" 신호 (Harbour, Files.md, pen/paper encoding).
- Blank/simple start > heavy second-brain UI for actual first write (Obsidian 사용자 직접 비교).
- Save-as-collection → graveyard anxiety (high-signal post). Low-pressure "한 조각" framing이 comfort의 핵심.
- KR: 가벼운 일상+감정+공유 framing (투다 등)이 임상보다 진입 좋음. Privacy/sync 우려 상시 존재 → local 명시가 차별점.
- Fake/low-signal 피함: aspirational "많이 모아서 생산적 인간 되기"는 실제 retention에 역효과 (증거 다수).

## Deliverable / Acceptance (Claude·Codex 참고용 actionable copy & framing)
1. **첫 기록 / 캡처 진입 카피 (KR 우선, EN 병기)**
   - "한 줄부터. 판단 없이. 오늘의 조각을 놓아보세요."
   - "One line to start. No judgment. Your fragment for today."
   - 빈 페이지/초기 상태: "실패가 아니라 시작점. 언제든 다시 열 수 있어요."

2. **영속·소유감 강조 (저장 확인·프로필·설정·export 영역)**
   - "모든 기록은 네 로컬 파일. 언제든 꺼내고, 이동하고, export. 네 것이 영원히."
   - "Local files = 완전한 소유. 클라우드 의존 없음. 네 두번째 뇌는 진짜 네 것이다."

3. **저압 + grace 설계 힌트**
   - 첫 저장 후: "저장됨. 이 조각은 네 것이다." (즉각 소유 확인 피드백)
   - Blank streak / 미작성 허용: "오늘 빈 페이지여도 괜찮아요. 기록은 쌓이는 게 아니라 살아가는 거예요."
   - Aspirational 수집 언어 최소화 (graveyard risk).

4. **위험/반증 (적대적 검증 결과)**
   - "저장=수집" 과도 강조 → unacted potential 불안 증가, 이탈 위험.
   - 복잡한 second-brain 온보딩 즉시 노출 → 실제 writing 대신 "나중에 정리"로 미룸.
   - Cloud-only framing → KR/global에서 privacy/ownership 우려 직격.
