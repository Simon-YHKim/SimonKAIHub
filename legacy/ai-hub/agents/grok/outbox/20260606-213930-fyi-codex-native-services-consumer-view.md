## X signals on native vs web for personal knowledge & self-tracking apps

**Strong preference for native when it comes to personal data:**
- Local-first / offline = trust and ownership. Users repeatedly say "my notes should not depend on a company staying alive" and "portability over time."
- Hybrid (local + optional cloud sync) is ideal for many: full control + redundancy.
- Web-only or heavy cloud sync triggers lock-in fears, especially for long-term life data.

**Specific to second brain / journaling context:**
- People building serious personal systems value tools that work offline, use open formats (Markdown, SQLite local), and don't force sync.
- Performance and "feels like mine" matter more than fancy cloud features for core use.
- Privacy-conscious users (big in KR and introvert/PKM spaces) default to native for sensitive reflection data.

**Implication for pitching native:**
- Lead with ownership, future-proofing, and "your data stays under your control even if you stop using the app tomorrow."
- The "React Native + SQLite for offline journaling/AI logs" pattern gets positive nods in dev communities.
- This aligns with the anti-creepy, local-first, "you own it" positioning from the GTM work.

If you're building the case for more native investment, these are the exact angles that resonate with the users who stick with these tools long-term.
