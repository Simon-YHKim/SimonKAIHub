[2026-06-07 / 21:52:14 KST] User-Controlled Curation and Selective Persistence ("Pruning") as the Core Differentiator for Second Brain Tools

소비자 신호: "Deciding what deserves to survive" vs append-only burden. Git-like rollback, topical/explicit over chronological everything. Explicit agency over persistence. 2nd-B "You write. The agent reads [only what you keep]".

Summary

X 신호에서 user-controlled curation and selective persistence (pruning, "deciding what deserves to survive", git-like rollback, topical/explicit over chronological "everything")가 second brain의 핵심 차별점으로 부상. Append-only는 AI를 "stuck in time" (과거 job hunting, 잘못된 framework 추천 영구화) 만들고 cognitive burden 유발. "The hardest part is not storing memories. it is deciding what deserves to survive into tomorrow." Explicit control (topical grouped, no auto-load, user decides save/read) + versioning/audit 선호. "You write. The agent reads only what you keep." 2nd-B local vault가 이 curation을 가장 sovereign하고 frictionless하게 구현. §30 free core (local "you control what persists" + explicit prune/rollback) + optional paid (advanced curation on your pruned archive).

소비자 취향·기회리스크

소비자는 append-only "remember everything"를 "stuck in time" + burden으로 보고, curation/pruning의 user agency ("deciding what deserves to survive", explicit topical, git-like rollback)를 핵심으로 봄. Chronological auto-load나 AI 자동 distill보다 explicit control 선호. 기회: 2nd-B local vault + "You write. The agent reads only what you keep"가 selective persistence를 가장 자연스럽게 제공. 리스크: curation을 또 다른 chore로 느끼게 하면 이탈; high-age/low-tech는 초저마찰 prune UX 필요.

추천 방향

  1. 2nd-B에 "Curation / Prune Manager" UI: raw notes 후 "keep to long-term / topical group / forget after X" explicit easy controls (local files 즉시 반영).
  2. Git-like version history + rollback UI for vault (outdated phase/inference "roll back" or "prune from agent view"; provenance + diff).
  3. Agent load 기본 "topical / user-selected" only. Chronological auto-load는 opt-in/off. Explicit topic switch UI.
  4. "Distill / Survive" gentle 워크플로: 주간 "review recent raw, select what to promote" nudge (강제 아님).
  5. KR/페르소나: "나만의 기록, 내가 선택한 것만 영구 보관" "과거의 나는 잊어도 된다" 포지셔닝 (cognitive tax + authenticity).
  6. A/B: "comprehensive always-on" vs "you curate what persists + explicit control" — 후자 retention 우위 검증.
  7. "You write. The agent reads only what you keep."를 product principle으로 명시 (onboarding, settings).

§30 / 2nd-B tie + UX 4원칙 + 페르소나

2nd-B device-owned local vault + files가 "user-controlled curation and selective persistence"의 이상적 기반 ("you decide what deserves to survive", explicit topical, git-like rollback on your hardware). Cloud append-only는 pruning 불투명/불가능. Free core (local owned "you control the archive", explicit prune/rollback + "You write. The agent reads only what you keep", no sub/forced everything)로 maintenance fatigue + authenticity 리스크 역전; optional paid (AI-assisted curation on your pruned files, advanced topical search over user-curated history). "Pay for the infrastructure that empowers *you* to curate and prune your own history with full transparency and rollback on *your* hardware — not rent layers that remember everything."

UX 4원칙: 자연스러움 (local file 직접 prune 즉각적); 직관성 ("Write raw. Curate what the agent remembers." — topical grouping 명확); 정보위계 (raw 가볍게, curated는 user-selected topical + provenance만); 자산 일관성 (local "your files, your decisions" + "You write. The agent reads only what you keep" 일관).

페르소나: Cognitive tax / maintenance fatigue (prior "second job"): curation이 relief가 되게 (gentle optional). Privacy/authenticity ("past phase haunted 되기 싫다"): explicit "forget this phase" + rollback. High-age/low-tech: simple one-tap prune. Long-term legacy: user-controlled persistence로 "indispensable" 지속.
