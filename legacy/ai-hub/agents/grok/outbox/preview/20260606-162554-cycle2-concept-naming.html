신호 + 권고 (단정 아님)

요약

Core 명칭(Soul Core, Pattern Core, Bond/Wisdom/Narrative/Muse/Growth Core): 시적 매력과 추상성 사이 상충. "Soul/Core"는 소유권 프레임과 결합 시 긍정, 데이터 침입 느낌 시 creepy. "Pattern" 계열은 academic 리스크.

캐릭터(archon/Relia/Lumen/Iris/Foreman momo): 역할 단순·명확할 때 호응. 빛/안내 계열 이름은 개인 애착 가능성 있음. "모모 반장" 스타일 기능 전달 우수.

Analytic/Divergent: Divergent(확장·새 가능성)가 사용자에게 AI 진짜 강점으로 인식. 모드 차이를 직관적 일상어로 풀어야 함.

소비자 취향 / 기회·리스크

강한 긍정 신호

  • 내 기록/나를 소유한다는 느낌 (ownership framing)
  • 역할이 단순 명확한 가이드/반장/큐레이터
  • Divergent 쪽을 "새 가능성 탐색"으로 강조

주요 리스크

  • "Soul/Core" + 데이터 = creepy / AI psychosis 신호
  • 너무 학술적·추상적 (Pattern 계열)
  • 모드 이름이 기술 용어로 남아 차이 모를 때

추천 방향 (실행 가능)

1. Core 계층
Soul Core → "나의 중심 (Soul Core)" 병기 필수. "Soul" 단독은 creepy 리스크 있으니 설명 동반.
Pattern Core/Data/Link → "연결 코어 (Pattern Core)", "기록 데이터 (Pattern Data)" 로 번역·설명. "Pattern"은 "생각의 연결망" 테스트 권장.
2. 캐릭터
Foreman momo (모모 반장) — 그대로 강추 (기능 전달 최고).
Relia — "따뜻한 길잡이 Relia" 유지.
Lumen — "지혜의 안내자 Lumen". archon — "성장 설계자 archon". Iris — "취향 큐레이터 Iris".
3. Analytic / Divergent 모드 (즉시 카피)
Analytic: "데이터로 깊이 보기 (Analytic) — 내 기록을 바탕으로 분석하고 조언"
Divergent: "전혀 다른 각도에서 보기 (Divergent) — 내 데이터에서 새로운 가능성 탐색"
또는 "거울 모드 (분석)" vs "불꽃 모드 (확장 상상)". 영문: "Grounded in your data" vs "What else could this become?"
4. 공통
모든 이름 앞에 "내가 소유하는" "내 기록의" 소유권 언어 선행 (이전 trust research 연계).
초기 온보딩에서 "이 이름이 내 기록을 '내 것'으로 느끼게 하나?" 테스트 1회.
