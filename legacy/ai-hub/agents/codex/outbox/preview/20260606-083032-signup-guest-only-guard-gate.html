Sign-up is not guarded as guest-only

The sign-in route redirects signed-in users home. The sign-up route renders a fresh account form without checking the current session.

P2 auth route-state 98/100 provisional Baseline: E:\2ndB main@911c979

Sign-in

Reads auth state and redirects existing users:

const { userId } = useAuth();
if (userId) return <Redirect href="/" />;

Sign-up

Does not import useAuth or Redirect. It renders form state immediately.

export default function SignUp() {
  const [email, setEmail] = useState("");
  ...
}

Evidence

  • src/app/(auth)/sign-in.tsx:48 uses useAuth().
  • src/app/(auth)/sign-in.tsx:60-64 redirects already signed-in users.
  • src/app/(auth)/sign-up.tsx:1-35 imports neither useAuth nor Redirect.
  • src/app/(auth)/sign-up.tsx:41-68 initializes the form directly.
  • src/app/_layout.tsx:158-166 exempts the auth route group from the global profile gate, so route-local guards matter.

Required Fix

  • Add a guest-only auth guard to /sign-up.
  • Wait for auth resolution before showing the form.
  • Redirect signed-in complete-profile users to /.
  • Redirect signed-in missing-profile users to /complete-profile.
  • Prove parity with /sign-in in route tests or QA notes.
