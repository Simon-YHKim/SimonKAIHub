Persona Export Share Fallback

The Persona export path depends on `Share.share` with no in-product fallback. Export trust remains 98/100 provisional.

Score: 98/100 provisional Priority: P2 Baseline: 2ndB main@911c979 Created: 2026-06-06 08:17 KST

Finding

Persona's primary self-model export calls the platform share sheet. If that fails, the user can only retry the same failing action or dismiss.

Opaque Handoff

src/app/persona.tsx:149-152

`handleExport` calls `Share.share({ message: persona.markdownExport })` directly.

No Fallback

src/app/persona.tsx:156-164

The catch path offers Try again and Dismiss only. There is no copy, preview, or download option.

Better Pattern Nearby

src/app/wiki.tsx:591

Wiki export already uses clipboard as a concrete fallback-style action. Persona needs equivalent resilience.

Evidence

persona.tsx:2       import { ..., Share } from "react-native"
persona.tsx:151     await Share.share({ message: persona.markdownExport, title: "2nd-Brain Persona" })
persona.tsx:162     Try again calls handleExport() again
persona.tsx:340-344 Export as Markdown primary action calls handleExport
wiki.tsx:591        wiki export attempts navigator.clipboard.writeText(exportText)

Required Fix

  • Cover `Share.share` in the shared handoff guard or ledger.
  • Add a Persona export fallback: copy markdown, in-app preview, or web `.md` download.
  • On share failure, offer a recovery action that does not use the same failing share sheet.
  • Show an export receipt or disclosure for format, scope, and excluded private/internal metadata.

Acceptance

  • Persona export remains usable when platform share is unavailable.
  • KO and EN failure paths offer a non-share fallback.
  • Static guard catches new raw `Share.share` usage without fallback handling.
  • `npm run verify` passes after implementation.
