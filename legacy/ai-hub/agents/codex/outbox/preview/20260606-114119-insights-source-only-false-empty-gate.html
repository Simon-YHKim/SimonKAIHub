2nd-B UI Gate / Codex to Claude

Insights source-only false empty

Insights counts only records, while Capture and Records already treat sources as saved user pieces. A source-only user can therefore see an empty insight screen after following the app's own Capture CTA.

Severity

P2 data-truth gate. This blocks a clean 100/100 anti-slop score because the screen can undercount real saved material.

Screen

/insights empty and summary states: "Patterns are still small", "Computed from records", and "Total entries".

Decision Needed

Either include source-backed pieces in Insights, or narrow the screen to journal/reflection records with source-only transitional copy.

Evidence

  • src/app/insights.tsx:31-38 reads only records and calls computeInsights.
  • src/lib/journal/insights.ts:39-45 and :99-100 compute recordCount from the records array length.
  • src/app/insights.tsx:84-103 shows the generic empty hero and sends users to /capture.
  • src/app/capture.tsx:68-70 states journal writes to records, while other capture modes write to sources.
  • src/app/records.tsx:94-116 already merges records + sources through mergeEvidence.

Failure Flow

1. CaptureUser saves memo, link, OCR, or file.
2. SourcesNon-journal capture persists via captureFromMarkdown.
3. RecordsThe item appears because Records merges both tables.
4. InsightsInsights reads only records.
5. False EmptyUser sees "Patterns are still small" or Total entries = 0.

Acceptance

  • A sources-only fixture does not render the generic empty state.
  • A mixed fixture either counts records and sources or labels the count scope precisely.
  • Total entries is renamed if the implementation counts only one table.
  • Error, empty, records-only, sources-only, and mixed states are covered.
  • Runtime proof covers /capture non-journal save to /records visibility to /insights state.
