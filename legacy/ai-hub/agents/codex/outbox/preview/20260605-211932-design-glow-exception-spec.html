DESIGN.md Signal-Glow Exception for Premium Shared Surfaces

Prepared by Codex for the shared-surfaces consistency fix. Scope is documentation only: no application code changes are included in this artifact.

Decision

Sanction one narrow glow family: named, colored signal-halo glows using the design tokens soulViolet and mint, only for art, aura, and ambient signal moments in premium backgrounds.

Remove black elevation shadows: any #000, black, or opacity-black box-shadow/drop-shadow used to lift dark cards, glass panels, shells, controls, or shared surfaces must be removed from premium shared components.

This resolves the apparent conflict between the shadow/glass/gradient audit finding and the intentional premium art direction: colored aura is a content signal; black shadow elevation on dark UI is not.

Sanctioned Glow

Name Tokens Allowed Use Constraints
signal-halo.soulViolet rgba(var(--soul-violet-rgb), 0.18) to 0.28 Ambient premium background aura, art-only edge bloom, non-interactive hero signal. No layout elevation. No text container glow. No card shadow. Must be behind content or clipped to decorative art.
signal-halo.mint rgba(var(--mint-rgb), 0.12) to 0.22 Secondary signal accent paired with violet in premium background art. Use sparingly. Mint may not become the dominant page wash. Must not imply focus, success, or active state unless the component explicitly owns that semantic.
signal-halo.mixed Layered soulViolet and mint only Soft multi-stop aura in premium/background.tsx and decorative brand art. Maximum two colored halo layers per component. No blur value above 72px on mobile. Prefer static gradients over runtime blur filters on Android.

Disallowed Shadow

Pattern Decision Replacement
box-shadow: 0 ... rgba(0, 0, 0, ...) Remove from dark shared surfaces, including premium cards, panes, sheets, shells, and glass panels. Use border: 1px solid var(--surface-border) plus a subtle top highlight token if needed.
filter: drop-shadow(... #000 ...) Remove. It is visually muddy on dark surfaces and expensive when combined with blur/elevation on Android. Use no filter. If separation is required, adjust surface fill contrast by one token step.
Glass panel with backdrop blur plus black shadow Remove the shadow. The blur and border already define the material. Use --surface-glass-fill, --surface-border, and --surface-highlight.
Large black ambient shadow under premium art Remove unless it is a literal image asset shadow from the artwork itself. Use sanctioned signal-halo tokens only when the glow is part of the art direction.

Exact DESIGN.md Wording

Add the following section to DESIGN.md under visual effects, premium surfaces, or the shared component rules.

### Signal-Glow Exception

Premium shared surfaces must not use black elevation shadows on dark UI. Remove `#000`, `black`, and opacity-black `box-shadow` or `drop-shadow` effects from dark cards, glass panels, shells, controls, and shared surface components. Depth on dark surfaces comes from fill contrast, `--surface-border`, `--surface-border-strong`, and the optional top-edge `--surface-highlight` token.

The only sanctioned glow exception is the named `signal-halo` family. `signal-halo` is a colored art effect, not an elevation model. It may use only `soulViolet` and `mint` tokens, and only for premium background aura, decorative brand art, or non-interactive ambient signal. It must not be applied to text containers, cards, buttons, menus, sheets, dialogs, or other UI surfaces as a substitute for elevation.

Recommended signal tokens:
- `--signal-halo-soul-violet: rgba(var(--soul-violet-rgb), 0.24)`
- `--signal-halo-soul-violet-soft: rgba(var(--soul-violet-rgb), 0.16)`
- `--signal-halo-mint: rgba(var(--mint-rgb), 0.18)`

Recommended surface separation tokens:
- `--surface-border: rgba(255, 255, 255, 0.10)`
- `--surface-border-strong: rgba(255, 255, 255, 0.16)`
- `--surface-highlight: rgba(255, 255, 255, 0.06)`
- `--surface-glass-fill: rgba(255, 255, 255, 0.045)`

Implementation rule: `src/components/premium/background.tsx` may retain or migrate colored premium aura to `signal-halo` tokens when the glow is decorative art. `src/components/premium/surfaces.tsx` and other shared UI surfaces must remove black shadows and use border, fill, and highlight tokens for separation. On Android, prefer static gradient fills and lower blur radii over backdrop blur plus shadow stacks.

Follow-On Fix Scope

  • Keep: colored violet and mint premium background halos when named as signal-halo and constrained to decorative art.
  • Remove: black drop shadows from surfaces.tsx cards, panes, glass wrappers, and reusable premium surface primitives.
  • Replace: visual separation with surface border, fill, and highlight tokens.
  • Android: avoid combining backdrop blur, large blur filters, and black shadows on the same dark surface.
