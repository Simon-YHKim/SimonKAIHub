P1 Findings

P1 Capture First View

/capture should be the lowest-friction raw entry surface. It currently asks the user to process modes, track, gates, format management, daily prompt, tags, and graph CTA.

  • Keep one primary input and one primary submit action above the fold.
  • Move mode/format/track controls into compact secondary areas.
  • Remove subscription coming-soon copy from the core path.

P1 Source Lifecycle

/inbox and /wiki split Phase 1 and promotion feedback across alerts, row buttons, and duplicated actions.

  • Show Raw, Summarized, Promoted, Linked, Stale chips.
  • Replace Phase 1 alerts with inline recoverable panels.
  • Use one primary row action plus overflow menu.

P1 Wiki Density

/wiki mixes browse, export, graph stats, Phase 1, Jarvis, graph handoff, and delete. It also still links to /journal.

  • Move export into a sheet or drawer.
  • Move destructive delete behind secondary action.
  • Replace /journal links with /capture.

P1 Chat Layout

/jarvis and /interview need a composer-safe shell. Chat routes should not share the same vertical assumptions as list pages.

  • Compact header after first turn.
  • One segmented mode hierarchy.
  • Quick chips remain secondary to composer.
  • Interview adds progress and review-before-save.

P2 Findings

Screen Issue Requested Direction
/records Filters still include legacy journal and imagine labels. Map legacy data to current labels and show provenance in current product language.
/import Success promotes graph navigation before source review. Show extracted title, type, tags, confidence, and save/review action before graph.
/formats Absolute toast uses raw bottom spacing, independent of shell clearance. Use shared toast/shell clearance, put schema and delete in contained sheets/menus.

Implementation Order

  1. Shared shell clearance and route metadata from the navigation shell request.
  2. Simplify /capture first viewport and remove subscription coming-soon copy.
  3. Add source lifecycle chips and inline Phase 1 result pattern to /inbox and /wiki.
  4. Contain /wiki export/delete actions and sweep /journal links.
  5. Apply composer-safe chat layout to /jarvis and /interview.
  6. Clean /records, /import, and /formats taxonomy/review/toast details.

Release Acceptance

  • /capture first viewport has one primary input and one primary submit action.
  • Phase 1 output is inline and recoverable, not alert-only.
  • No user-facing /journal links remain in /wiki.
  • Chat composer never collides with keyboard or bottom navigation.
  • Legacy imagine and route-era labels do not appear as current IA labels.
  • No arbitrary bottom padding like 110, 120, or raw spacing.xl remains in audited route shells.
