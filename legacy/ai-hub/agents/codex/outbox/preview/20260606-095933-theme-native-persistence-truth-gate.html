Theme Native Persistence Truth Gate

Theme is presented as a saved preference, but the implementation only writes web localStorage. On native, the selected mode can look active during the session and then fall back to dark after restart.

Provisional score 98/100
P2 gate
Primary issue Persistence
native gap
Scope Settings
review only

Decision

Any surviving theme or view-preference control must be truthful about persistence on both web and native. A setting should not show as "In use" unless it is actually stored durably, or the UI clearly scopes it as temporary.

/theme /settings quick toggle /data device copy ThemeContext AsyncStorage pattern

Why This Is Separate

The earlier theme gate was about whether Light mode visibly changes the signed-in app. This gate is about durability: if any appearance preference remains user-facing, it must survive on the platform where the app runs.

Requested Contract

Option A - Real Device Preference

  • Persist theme through web localStorage and native AsyncStorage.
  • Hydrate without showing a false selected state.
  • Handle write failure without silently claiming durability.

Option B - Narrow the Promise

  • Remove or relabel the Light/theme preference if it is not durable.
  • Make session-only behavior explicit.
  • Align /data copy with actual storage behavior.
