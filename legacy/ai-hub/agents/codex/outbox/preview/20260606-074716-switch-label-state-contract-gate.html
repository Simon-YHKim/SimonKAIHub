Switch Label/State Contract Gate

Privacy and consent switches need a governed semantic contract. Adjacent row copy is not enough when the switch itself is a focus target.

Score: 98/100 provisional Priority: P2 Target: Claude implementation Created: 2026-06-06 07:47 KST

Finding

The UI visually pairs labels with switches, but raw switches do not consistently expose label, description, locked reason, checked state, or pending state.

Privacy Preferences

src/app/privacy.tsx:162-188

Rows include label, description, locked badge, and a bare Switch. The semantic contract is not owned by a component.

Consent Dialog

src/components/consent/ConsentDialog.tsx:44-50

Two sensitive testimonial consent switches rely on nearby text rather than explicit labels and hints.

Better Local Patterns

ConsentNotice.tsx, formats.tsx

ConsentNotice exposes role/state on the whole row. Formats at least labels the share switch. These should become a shared contract.

Contract Expected

External analytics
Label + description + checked state are exposed together.
Public sharing
Disabled reason is exposed when minors or policy locks prevent changes.

Required Fix

  • Add a shared SwitchRow / PreferenceToggle component.
  • Use it in privacy preferences and testimonial consent.
  • Expose label, description, checked state, disabled reason, and pending/busy state.
  • Migrate format share toggle or document it in an exception ledger.
  • Add a guard for raw <Switch> usage outside the shared component.

Acceptance Proof

Privacy switches expose localized KO/EN labels and descriptions.
Minor-locked switches expose a disabled/locked reason.
Privacy save failure is announced or associated with the changed preference.
Testimonial consent switches expose localized KO/EN labels and hints.
Formats share switch keeps format name, checked state, and pending state.
Raw Switch scan or exception ledger is included.
