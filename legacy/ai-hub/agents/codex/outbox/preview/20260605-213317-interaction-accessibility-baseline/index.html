Interaction & Accessibility Baseline

전체 화면 UI 개선 목표의 다음 라운드입니다. 이번 감사는 화면 장식이 아니라 사용자가 실제로 누르고, 입력하고, 로딩을 기다리고, 오류를 회복하는 계약을 정리합니다.

Pressable Hits
112

Route-local interaction이 많아 contract drift 위험이 큽니다.

Blank Loading Routes
19

Auth/loading 중 빈 화면을 반환하는 route 수입니다.

Priority
Shared

화면별 패치보다 공통 primitive가 먼저입니다.

Findings

P0

No blank auth/loading routes

19개 route가 if (loading) return null 또는 if (authLoading) return null을 사용합니다. 사용자에게는 앱이 멈춘 것처럼 보일 수 있고, 보조 기술에는 상태가 전달되지 않습니다.

Claude action: shared RouteLoadingGate 또는 InlineLoader를 route-wide로 적용하고 accessible status text를 붙이세요.

P0

Route-local Pressables need contracts

Auth, Wiki, Inbox, Jarvis, Graph empty card, Capture에 직접 구현한 Pressable이 많습니다. 일부는 role/label/state가 없고, disabled/busy/selected 의미가 일관되지 않습니다.

Claude action: TextActionButton, ChipButton, SegmentedControl, InteractiveRow를 만들고 실제 명령형 Pressable을 교체하세요.

P0

Assessment scale controls

Big Five, MBTI, Attachment의 Likert 숫자 버튼은 hitSlop=2이고 radio-like label/state가 없습니다. 빈번하게 누르는 평가 UI라 우선순위가 높습니다.

Claude action: 5점/7점 공용 LikertScale을 만들고 44px target, selected state, question/value/accessibility label을 제공하세요.

P1

Placeholder-as-label forms

Input은 raw TextInput wrapper라 label/error/helper 구조가 없습니다. Auth, Capture, Import, Audit, Interview, Settings, Account, Wiki Template Editor에서 placeholder가 실질 label 역할을 합니다.

Claude action: visible label, helper, error, required/optional, invalid state를 포함한 FormField를 추가하세요.

P1

Micro targets below 44px

Inbox action button은 minHeight: 30, Jarvis mode/modal buttons는 36px 수준, Capture tag controls는 22-28px glyph 기반입니다.

Claude action: minimum interactive target token 44px를 만들고 chip/text/modal/nav controls에 적용하세요.

P1

Modal and drawer semantics differ

Core Brain drawer, Jarvis modal/drawer, premium feedback modal, consent dialog가 서로 다른 backdrop/card semantics를 씁니다.

Claude action: shared AppModal/BottomDrawer로 backdrop close label, visible close, modal semantics, escape/back handling을 통일하세요.

P2

Error and empty states need action contract

일부 load failure는 alert에 의존하고, empty CTA는 아직 /journal로 가거나 화면마다 다르게 동작합니다.

Claude action: ScreenState variants를 표준화하고 loading/empty/error/retry/support/export escape를 route contract와 연결하세요.

Implementation Order

  1. RouteLoadingGate로 19개 blank route 제거.
  2. ChipButton, TextActionButton, InteractiveRow, SegmentedControl 추가.
  3. Big Five, MBTI, Attachment를 공용 LikertScale로 통합.
  4. Auth/capture/import/destructive form에 FormField 적용.
  5. Jarvis/Core Brain/Consent/Premium feedback modal contract 통일.
  6. Inline error/empty state pass.
