Quant Intro Storage Flicker Gate

A saved "skip this intro next time" preference should not briefly show the intro card before the assessment starts.

Project: 2nd-B Head: 911c979 Score: 98/100 provisional Priority: P2

Current Risk

visible starts as true, so the modal can render before AsyncStorage confirms the saved skip preference.

Affected Screens

Live assessment consumers are /big-five and /attachment. The component also still carries an mbti tool key.

Correct Shape

Use an explicit checking, show, skip decision. Render no intro card until the decision is known.

Evidence

src/components/quant/QuantIntroModal.tsx:36 function useShouldShow(...) src/components/quant/QuantIntroModal.tsx:37 const [visible, setVisible] = useState(true); src/components/quant/QuantIntroModal.tsx:41 AsyncStorage.getItem(storageKey) src/components/quant/QuantIntroModal.tsx:43 setVisible(seen !== "1"); src/components/quant/QuantIntroModal.tsx:87 if (!visible) onStart();

State Flow

User chose skip

Preference is stored as quant-intro-seen:*.

Route opens

Current code assumes modal visible before storage has answered.

Card flashes

The UI contradicts the saved preference for one render.

Storage resolves

visible becomes false.

Assessment starts

Expected outcome, but reached after a visible mismatch.

Acceptance

  • Do not initialize the storage decision as visible before AsyncStorage.getItem() resolves.
  • Represent decision state explicitly as checking, show, or skip.
  • Call onStart() once after commit when the decision is skip.
  • On storage failure, intentionally choose show after the failure is known.
  • Provide QA proof that repeated visits to Big Five and Attachment do not flash the intro card.
