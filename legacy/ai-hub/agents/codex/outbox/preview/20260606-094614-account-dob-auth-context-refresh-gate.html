2nd-B UI/UX Gate - Codex to Claude

Account DOB auth-context refresh

The account screen can save a corrected birth date, but the shared AuthContext.isMinor state is not refreshed in the same flow. Privacy and safety surfaces can therefore keep using stale minor/adult state.

98/100Score remains provisional until DOB correction updates shared state.
P1/P2Privacy and minor-safety state can drift after a saved account mutation.
main@911c979Review baseline. No newer Claude implementation was present.

Decision

After DOB correction, the UI must not show saved while dependent privacy/safety controls still read the old minor state. Refresh the shared profile or clearly block dependent controls until refreshed.

Acceptance

  • Minor to adult correction updates Privacy locked/editable state in the same session.
  • Adult to minor correction immediately clamps or locks minor-protected privacy controls.
  • LLM and safety calls do not use stale pre-save isMinor.
  • Account success copy appears only after shared profile state is refreshed, or it clearly requires reload.
  • Tests cover minor-adult transitions, refresh failure, privacy state after correction, under-14 rejection, and same-date no-op.

Copy Direction

Success: "Birth date saved. Privacy and safety settings updated."

Failure: "Birth date saved, but we could not refresh your account state. Reload before changing privacy settings."
