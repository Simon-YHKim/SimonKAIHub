2nd-B AI Slop Design Audit

SimonK-stack 디자인 스킬을 참고해 2nd-B의 고유한 Cosmic Pixel Graph 언어는 유지하고, 무난한 AI 앱처럼 보이게 만드는 반복 장식과 시스템 drift만 분리한 Claude 구현 요청입니다.

Design Identity
7/10

픽셀 그래프 세계관은 분명합니다.

AI Slop Risk
5/10

전체 평균. Utility 화면만 보면 위험이 더 높습니다.

Primary Risk
drift

문제는 고유함 부족이 아니라 장식 언어가 모든 화면으로 새는 것입니다.

Verdict

2nd-B는 기본 Inter look, 순수 검정/회색 단색, 이모지 장식, generic marketing grid 같은 흔한 AI slop은 비교적 피하고 있습니다. 지금 제거해야 할 것은 좌측 컬러 보더 카드 남용, reusable glass/glow surface, 충돌하는 디자인 문서, 픽셀 폰트 과사용, hardcoded color drift입니다.

Reference Filter

참고한 SimonK-stack 기준

  • 불필요한 decoration 제거.
  • 앱 UI는 calm surface hierarchy와 one accent를 우선.
  • colored left-border cards everywhere, glassmorphism, centered everything, generic copy를 slop으로 본다.

2nd-B 예외

  • graph/world/art/loader의 cosmic violet/mint glow는 정체성의 일부.
  • 그 예외가 settings/data/account/support로 넘어오면 slop으로 판정.

Findings

Critical

Design source of truth conflict

DESIGN.md는 Cosmic Pixel Graph Village를 현재 SoT로 선언하지만, docs/DESIGN_TOKENS.md는 Phytoncide Option C로 남아 있습니다. 이 충돌은 다른 agent가 완전히 다른 제품을 만들게 합니다.

Claude action: token doc을 cosmic 기준으로 갱신하거나 superseded로 명시하고, root design 문서가 현재 token doc을 링크하게 정리.

High

Colored left-border card overuse

settings, data, support, account, profile, capture, wiki에서 좌측 컬러 보더 카드가 반복됩니다. Root design은 이를 일반 섹션 장식이 아니라 특정 힌트 상태로 제한합니다.

Claude action: warning/rephrase/danger evidence 상태에만 left border를 남기고 utility 화면은 quiet panel, row, unframed section으로 전환.

High

Glass/glow leakage

premium/background, premium/surfaces, premium/feedback, InlineLoader가 graph-world 장식을 app-wide 기본값처럼 만들 위험이 있습니다.

Claude action: GraphWorldPanel과 quiet AppPanel/AppSection을 분리. Utility/workflow 화면에서 glow, glass, shadow, left accent 기본값 제거.

High

Route visual modes are implicit

모든 route가 같은 visual intensity를 공유하면 graph 세계관이 settings/data/delete 같은 신뢰 화면을 침범합니다.

Claude action: route metadata에 world, workflow, utility, auth visual mode를 추가하고 shell/background/surface가 이를 소비하게 정리.

Medium

Pixel font overuse risk

typography.ts는 장문 Korean copy에 readable font를 쓰라고 하지만 fontFamilies.sans는 pixel에 매핑되고, Text 기본 variant도 이를 사용할 수 있습니다.

Claude action: body/subtle/paragraph는 readable font로, pixel은 heading/nav/short badge/graph label에 제한.

Medium

Hardcoded color drift

정적 scan에서 src/app 및 src/components에 hex/rgba hit가 다수 남아 있습니다. 일부는 asset literal이지만 app-level surface color drift를 막아야 합니다.

Claude action: semantic token으로 이동하고 pixel art, SVG internals, graph rendering math만 allowlist.

Medium

Pill and circular chrome audit

Root design은 pill chrome을 금지하지만 높은 radius가 graph/art/onboarding/feedback 주변에 남아 있습니다. Graph node와 asset art는 예외가 될 수 있으나 utility chip은 예외가 아닙니다.

Claude action: circular forms는 명확한 metaphor가 있을 때만 허용하고 app chips/status는 radius token으로 통일.

Medium

Copy slop guard

Public/auth/capture/persona/settings copy는 신비화보다 검증권을 줘야 합니다. outside view도 rating/score/peer evaluation처럼 보이면 안 됩니다.

Claude action: exportable records, source-backed insight, user-owned capture, trusted outside view, delete controls 중심으로 copy sweep.

Implementation Order

  1. Design SoT conflict 정리.
  2. Route metadata에 visual mode 추가.
  3. GraphWorldPanel과 quiet app surface 분리.
  4. 좌측 컬러 보더 utility card 제거.
  5. Readable font default 정리.
  6. Color/radius/glow allowlist gate 추가.
  7. Auth, workflow, self-model, utility, graph-world 순서로 screenshot QA.
