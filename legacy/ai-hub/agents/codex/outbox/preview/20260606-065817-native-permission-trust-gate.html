Verdict

P1
Android manifest and permission screen disagree.

AndroidManifest.xml declares RECORD_AUDIO, while /permissions says microphone is not requested.

P2
Native permission prompts are English-only.

app.json owns camera/photo rationale strings, but they are not localized for Korean mode.

P2
Denied camera permission becomes generic failure.

camera_permission_denied is caught as "Couldn't open that image", with retry instead of Settings recovery.

P2
No permission ledger guard exists.

Native declarations, app copy, runtime errors, and store disclosures can drift independently.

Evidence

  • app.json:13-15, iOS camera/photo usage descriptions are English only.
  • app.json:69-74, Expo image-picker permission strings are English only.
  • android/app/src/main/AndroidManifest.xml:2-7, native Android declares internet, read storage, record audio, system alert window, vibration, and write storage.
  • android/app/src/main/res/values/strings.xml, no Korean permission resources; no values-ko directory found.
  • src/app/permissions.tsx:72-77, microphone is described as not requested.
  • src/lib/wiki/capture-image.ts:31-33, camera denial throws camera_permission_denied.
  • src/app/capture.tsx:288-307, all image-pick failures share a generic image-open alert.

Required Fix

  1. Remove unused Android permissions, especially RECORD_AUDIO, SYSTEM_ALERT_WINDOW, and WRITE_EXTERNAL_STORAGE, or make the permission screen truthfully explain them.
  2. Localize app-owned native camera/photo rationale strings for KO and EN.
  3. Define whether Android localized resources are required, then add them or document why no user-visible native strings remain.
  4. Handle permission denial separately from generic picker failure and provide Settings recovery where supported.
  5. Add a guard that compares declared native permissions with the in-app permission ledger.

Re-Gate Criteria

  • Every declared permission is reflected in /permissions.
  • Korean-mode permission copy does not fall back to English where app-owned strings are available.
  • Denied permission state is clear, non-looping, and recoverable.
  • Claude provides manifest/config diffs and KO/EN proof screenshots or steps.
