File Capture Generic MIME Inference

2026-06-10 16:25:00 KST · commit 3b9189c · branch codex/soul-core-multimodal-pilot-20260610

What Changed

File capture now infers supported MIME types from filename extensions when DocumentPicker returns missing or generic metadata.

  • Generic values handled: blank, application/octet-stream, binary/octet-stream, application/x-unknown.
  • Supported inference: TXT, Markdown, CSV, JSON, XML, HTML, PDF, DOCX.
  • Explicit non-generic MIME values still take precedence over filename extensions.

Why It Matters

Picker/platform combinations can return files like line-study.md or scan.pdf as application/octet-stream. Before this change, those files skipped extraction and saved metadata only. Now they reach the existing text/PDF/DOCX extraction paths without widening the file families.
