2nd-B UI Gate / Codex to Claude

Graph handoffs pass ids the graph cannot highlight

Record detail and wiki rows promise "See in graph", but they pass records.id or wiki_pages.id. The graph renders only sources.id shards, then silently ignores unknown highlights.

Severity

P2 interaction-truth gate. The action navigates but can produce no visible outcome.

Surfaces

/record/[id], /wiki, home graph /, and NavGraph highlight-on-return.

Root Cause

Three id namespaces are treated as one: records, wiki pages, and source graph shards.

Mismatch Flow

1. Detail ActionRecord detail sends highlightRecordId=row.id; wiki sends highlightWikiPageId=p.id.
2. Graph RouteHome graph accepts either param and calls it highlightId.
3. Data NodesGraph shards are built only from sources, with ids equal to sources.id.
4. Silent No-OpNavGraph returns when dataPositions lacks that id.

Evidence

  • record/[id].tsx:145-151: "See in graph" passes a record id.
  • wiki.tsx:329-330 and :945-959: wiki rows pass a wiki page id.
  • index.tsx:153-164: record/wiki highlight params are accepted as graph highlight ids.
  • index.tsx:252-280: graph nodes are created only from sources; records are queried only as a count.
  • NavGraph.tsx:931-938: unknown ids are ignored without feedback.

Acceptance

  • Record detail to graph always produces exact highlight, domain focus, or truthful fallback.
  • Wiki page to graph highlights the source-backed page or explains that entity/concept pages are not drawn as graph dots yet.
  • Unknown first-party highlight ids never fail silently.
  • Tests cover record id, wiki page id, source id, menu id, and missing id.
