What Changed

OCR and direct inline image calls now strip MIME parameters before allowlist and signature checks.

  • image/jpg; charset=binary normalizes through the OCR picker path to image/jpeg.
  • IMAGE/PNG; charset=binary normalizes through direct callGemini inline image preflight to image/png.
  • Signature sniffing and MIME compatibility checks still run after normalization.

Why It Matters

Some picker/browser APIs include MIME parameters. Valid image bytes should not fail before OCR just because the declared MIME has a harmless parameter. This keeps capture OCR and lower LLM image boundaries aligned.
