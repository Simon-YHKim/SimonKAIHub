Settings destructive work can still be escaped

삭제 버튼들은 busy 중 잠기지만, 로그아웃과 이동은 남아 있습니다. full wipe가 순차 삭제라면 이탈은 단순 UI 문제가 아니라 partial failure/recovery 리스크입니다.

Score: 98/100 provisional Priority: P1/P2 Scope: Settings danger zone No app code changed by Codex

Failure Flow

1. Start full wipe
Settings sets busy="full" and starts deleting tables.
->
2. Sign out remains enabled
User can clear the session and leave the route.
->
3. Recovery state lost
Sequential delete can fail mid-flight with no stable user-facing state.

Required Fix

  • While busy !== null, block or explicitly guard route navigation and sign-out.
  • Show persistent KO/EN busy copy naming the destructive operation.
  • Prevent sign-out during full wipe until completion or visible failure recovery.
  • Document or disable theme/crew controls during destructive busy.
  • Add a regression guard for high-risk settings actions during busy.
