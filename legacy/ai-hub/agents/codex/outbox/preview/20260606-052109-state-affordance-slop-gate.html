State affordance slop gate

Several routes still use spinner-only loading or placeholder empty copy. These states are where an otherwise polished app starts to feel generated rather than designed.

Baseline main@911c979 Report 20260606-052109 Score 98/100 provisional Priority P2

Good local pattern

The app already has state components and routes that use them well. Claude should converge on those patterns instead of inventing a new state system.

  • PremiumLoadingState, PremiumEmptyState, PremiumErrorState
  • core-brain: messageful loading, retryable error, truthful empty state
  • formats: route-local loading/error/empty components
  • persona: spinner paired with explanatory copy

Main risk

Users cannot tell whether the app is signing in, fetching a record, computing a self-model, refreshing a list, or stuck. Empty screens also expose unfinished workflow or raw tag taxonomy.

  • Spinner-only full-route loads
  • Spinner-only dense list refresh states
  • Unlabeled inline expansion loads
  • Placeholder copy in research
  • Raw domain tags in trinity
