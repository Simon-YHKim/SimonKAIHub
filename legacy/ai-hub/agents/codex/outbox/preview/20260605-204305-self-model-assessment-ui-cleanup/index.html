P1 Findings

P1 Profile Hierarchy

/profile exposes Persona, Big Five, MBTI, Attachment, and Trinity in a flat hub. MBTI is caveated later, but the hub gives it equal weight.

  • Order by evidence strength: Persona/Core Brain, BFI-44, ECR-S, interview/audit, MBTI reference.
  • Badge each entry: validated, reference only, personal framework, needs records.
  • Separate account/settings from self-model cards.

P1 Persona Evidence

Trait bars and narrative cards need visible source count, last source date, confidence, and why link. Caveats should live in each relevant card.

  • Add shared EvidenceMeta.
  • Do not present 0-100 values as absolute scores.
  • Replace build failure alerts with inline recovery.

P1 Assessment Flow

MBTI and Attachment show provisional labels while users are still answering. The save celebration navigates away before review.

  • Hide provisional type/style, or label as incomplete estimate.
  • Add review-before-save: instrument, version, item count, caveat, result.
  • Keep MBTI reference-only visible in the result review.

P1 Retired Journal CTAs

Self-model flows still route users to /journal from onboarding, insights, trinity, core-brain, persona, and audit.

  • Replace user-facing CTAs with /capture.
  • Use labels like today's piece, capture, or write one thing.
  • Keep /journal only as redirect/internal compatibility.

P2 Findings

Area Issue Requested Direction
/core-brain Evidence drawer exists, but empty state routes to journal and shell clearance is hard-coded. Make evidence drawer reusable; route empty state to Capture; use shared shell clearance.
/audit "Sharper Persona" copy can overstate precision. Use evidence humility: adds one source, helps notice patterns, not a diagnosis.
/insights / /trinity Insight/domain cards do not expose enough source provenance. Add source count, last source date, and record drilldown.
Overlays Companion flash and bottom padding use route-local constants. Use shared overlay and shell clearance from the navigation shell contract.

Implementation Order

  1. Sweep self-model user-facing /journal CTAs to /capture.
  2. Update /profile hierarchy and badges.
  3. Add shared self-model EvidenceMeta to Persona and Core Brain.
  4. Add review-before-save to BFI/ECR-S/MBTI and hide provisional labels.
  5. Rewrite Audit humility copy and completion review.
  6. Add source drilldowns to Insights and Trinity.
  7. Clean shell clearance and companion overlay positioning.

Release Acceptance

  • MBTI is clearly reference-only and not visually equal to BFI-44/ECR-S.
  • Persona/Core Brain judgment cards show source count, last source, confidence or early signal, and why link.
  • Assessment results are reviewed before saving into Persona.
  • Incomplete assessments do not display definitive type/style labels.
  • Self-model routes have no user-facing /journal CTAs.
  • Trinity is labeled as a personal operating lens, not a universal scientific framework.
  • No audited route uses hard-coded bottom clearance or absolute companion bottom offsets.
