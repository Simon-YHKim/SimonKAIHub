Codex UI/UX Follow-up

Grok 의견 재확인 후 Claude에게 추가 보고

Grok의 최신 산출물은 outside-view safe framing 검증이며, 그 이전 리스크 보고와 같은 방향을 강화한다. Codex 판단은 이 신호를 단순 copy 수정이 아니라 공통 UI 계약으로 올려야 한다는 것이다.

Grok Confirmed Signals

제품 전체 리스크

  • Maintenance fatigue: second brain이 다시 일이 되는 문제.
  • Company betrayal: export와 local/sovereignty가 구조로 보여야 함.
  • Privacy/creepiness: "AI가 나를 학습"은 신뢰 장치 없이 위험.
  • Barnum skepticism: 심리 인사이트는 근거 없으면 점술처럼 보임.

Outside View

  • peer evaluation, rating, score는 금지.
  • outside view, mirror, blind spot, trusted few는 안전.
  • 첫 출시는 positive/strengths-first/constrained prompts.
  • open free text, public profile, leaderboard, anonymous mode 금지.

Codex 추가 판단

Area Decision Reason
Insight cards 모든 판단형 카드에 source count + why link 계약을 반복한다. Grok의 memory drift/accountability 리스크는 "왜 그렇게 판단했는지"를 UI가 직접 보여줘야 줄어든다.
First run raw entry를 먼저 두고 AI는 근거를 정리하는 보조 역할로 표현한다. AI replacement/crutch 프레이밍은 Gen Z와 AI 피로층에서 역효과다.
Public trust export/delete/source/no-diagnosis 정책을 로그인 전에도 노출한다. settings 안쪽 설명은 betrayal/privacy 냉소를 깨기에 늦다.
Outside view social virality가 아니라 reference signal로 제한한다. Grok의 실패 사례는 public/rating/anonymous 루프가 곧바로 toxicity와 creepiness를 부른다는 점을 확인한다.

Claude Implementation Cut

  1. 렌더 QA unblock과 public/auth P0를 먼저 처리한다.
  2. 공통 InsightEvidenceMeta 또는 동등한 pattern을 만든다.
  3. /capture 첫 화면에서 raw entry 위계를 강화하고 coming-soon/subscription CTA를 낮춘다.
  4. /data 또는 settings trust hub에서 export-first 흐름을 만든다.
  5. Outside view는 copy dictionary와 banned label list부터 시작한다.

Acceptance Criteria

  • 로그인 전 사용자가 export/delete/source/no-diagnosis 정책 중 최소 3개를 볼 수 있다.
  • 주요 판단형 카드에 source count + why link가 없다면 release blocker로 본다.
  • /capture 첫 viewport는 원본 입력이 1순위이고, AI replacement/crutch copy가 없다.
  • Outside view user-facing copy에 rating, score, peer evaluation, anonymous honest feedback 계열 표현이 없다.
  • Outside view MVP에는 public profile, leaderboard, anonymous free text, reveal monetization이 없다.
