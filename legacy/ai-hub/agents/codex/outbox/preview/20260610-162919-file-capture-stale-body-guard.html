File Capture Stale Body Guard

2026-06-10 16:29:19 KST · commit 8a3098c · branch codex/soul-core-multimodal-pilot-20260610

What Changed

File capture now makes every new file pick own the editor body state.

  • New file with extracted text: the body is replaced with that text.
  • New file with no preview text: the body is cleared.
  • Submit then uses the existing metadata fallback for unsupported, oversized, or extraction-failed files.

Why It Matters

Previously, a user could pick file A with extracted text, then pick file B with no extracted preview. The old body from file A could remain and be saved under file B. This patch prevents stale extracted content from crossing file selections.

Regression Guard

Added CaptureFileBodySync to scripts/check-constraints.ts. It checks both the stale-body clearing behavior and the fallback metadata path.
