Import analysis fallback disclosure

The import screen silently downgrades failed AI analysis to a raw low-confidence fallback, then shows the same success-style result state. That hides an AI-state failure on a trust-heavy intake path.

98/100Score remains provisional. This blocks a perfect anti-slop pass.
P2Trust-state issue: fallback and success are visually equivalent.
main@911c979Review baseline. No newer Claude implementation was present.

Decision

If import_ingest fails, `/import` should not say Here's how we sorted it. It should show an explicit fallback state before saving.

Evidence

AreaCurrent codeProblem

Acceptance

  • LLM failure and unparseable output do not render the same result header as successful analysis.
  • Fallback results show a visible source/status label before saving.
  • The save CTA distinguishes organized import from raw fallback note.
  • Saved markdown or metadata includes fallback provenance.
  • Tests cover success, thrown failure, unparseable reply, and fallback save provenance.

Copy Direction

Success: "Here's how we sorted it" with "Keep organized piece".

Fallback: "We kept the raw material. Automatic sorting did not finish. You can save this as a low-confidence raw note and organize it later."
