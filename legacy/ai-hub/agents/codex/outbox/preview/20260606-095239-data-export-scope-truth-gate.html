Data Export Scope Truth Gate

100/100 anti-slop review for the data portability path across Support, Data, Settings, and Wiki.

Project: 2nd-B App: main@911c979 Priority: P1/P2 trust Created: 2026-06-06 09:52:39 KST

Decision

The app must align every export and backup claim. Current UI promises deletion backup and file download, while implementation provides a wiki/sources markdown context bundle as selectable/copied text, with page-body truncation.

Score remains 98/100 provisional until this becomes truthful.

Evidence

Support FAQ

src/app/support.tsx:27 and :42 promise data as a file from Settings.

Problem: no Settings export action or file path is present.

Data Hub

src/app/data.tsx:48-70 says backup first, then routes export to Wiki text copy.

Problem: backup language is broader than the actual copy flow.

Settings Danger Zone

src/app/settings.tsx:316-317 and :531 tell users to export before destructive deletion.

Problem: the same screen deletes more data classes than the current export covers.

Wiki Export

src/app/wiki.tsx:306 uses bodyCharLimit: 4000; :589-629 provides copy/selectable text.

Problem: this is not a full downloaded file backup.

Export Model

src/lib/wiki/export.ts:1-7 and :111-168 export wiki pages and sources only.

Problem: records, account/privacy settings, and usage counters are outside the bundle.

Deletion Scope

src/lib/records/delete-bulk.ts:137-160 wipes wiki pages, sources, records, usage, self contexts, and templates.

Problem: "export before delete" reads as wipe-equivalent backup but is not one.

Required Product Choice

Option A - Full Export

  • Add a real export action from /data or /settings.
  • Cover every user-owned class affected by full wipe, or disclose exclusions.
  • Provide file download on web and share/save/copy fallback on native.
  • Show receipt: format, counts, generated time, exclusions, truncation.

Option B - Narrow Copy

  • Rename current flow to "Export wiki context" or "Copy knowledge context".
  • Remove "Export in Settings", "file download", and broad "your data" promises.
  • Tell users exactly what is not backed up before delete.
  • Show counts and the 4000-character per-page body limit in Wiki.

Acceptance

  • Support, Data, Settings, Wiki, and Manual agree on export location, format, and scope.
  • "Download as a file" appears only if a file/share-save path exists.
  • "Backup before delete" either covers the full deletion scope or lists exclusions.
  • Wiki export shows counts and truncation/exclusion status.
  • Static copy-contract checks prevent broad data export claims from drifting back.

Re-review Path

/support -> /data -> /wiki export -> /settings full wipe

A user must know exactly what is exported, where it happens, what format they get, and what will still be lost if they delete.
