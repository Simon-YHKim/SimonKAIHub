CI copy-law 룰 (2중벽)

user-addressed 마스코트 문자열에 대해: anti-presence(I'm here / 여기 있어요 / 같이 들여다 / 따뜻한 길잡이 / 동반자 / 친구처럼 금지) AND anti-oracle(미성년 심리 단정하는 bare-declarative 금지) — 그리고 출처 토큰(기록/records/patterns) + 사용자 통제 동반을 require. 기존 check-constraints.ts의 forbiddenTrustCopy 패턴 확장. "observational but humane"을 리뷰어 판단이 아니라 기계적으로 검사 가능하게.

여전히 Simon/counsel 에스컬레이션 (persona-sim 범위 밖)

  • ① 최우선: gadi.systemHint는 정적 문자열이 아니라 LLM 프롬프트 — 11pm 미성년에 대한 모델의 live 출력 안전은 copy-review로 게이트 불가. 기존 미성년 crisis 라우팅 + lexicon이 Relia의 관계 발화를 충분히 가두는지 = safety-eng + counsel 질문.
  • ② 미성년 제품에 named 마스코트 유지 자체(observational 재voice 후에도) = 구조적 결정. persona-sim은 재voice가 리스크를 줄임을 확인했을 뿐 제거하진 않음. "미성년엔 캐릭터를 무명 UI 어포던스로" 옵션은 이 게이트 위.
  • ③ es/pt/id 로케일은 동일 재voice + 네이티브 검수 필요("sourced+humane이 차갑지/oracle로 번역되지 않는지"). 이번 검증은 KO/EN만.
  • ④ (기존 D-21) SB243 classification memo · DPIA에 de-voiced register=완화책 기록.
