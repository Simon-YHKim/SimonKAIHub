미성년 AI-리터러시 ENABLEMENT — 제한이 아니라 학습자 면허

5렌즈 워크플로(2nd-B 코드 + AI-리터러시 교육학 + scaffold-vs-prohibition + 법규 + 실위험) grounded · 6 agents / 147 tool-calls / ~807K tokens · 2026-06-14 KST
가장 중요한 발견: 지금 2nd-B 코드에는 "미성년에게 AI를 끄는" 역량 제한이 사실상 존재하지 않는다. isMinor(<18)는 LLM 파이프라인 전체에서 단 하나 — 위기 핫라인을 청소년전화 1388로 라우팅하는 데에만 닿는다(safety.ts:266, classifier.ts:58). 모델·프롬프트·RAG·페르소나 깊이는 14세나 40세나 바이트 단위로 동일하다. 나머지 미성년 조건부 동작은 전부 데이터·웰빙 레일(가입 하한·egress 잠금·광고/training 차단·위기 라우팅)이고, 이건 아이의 데이터를 지킬 뿐 AI가 무엇을 가르칠지는 안 건드린다. → 풀 게 없다. "보조바퀴(scaffolding)"는 얹었다 빼되, "자전거(AI 역량)"는 절대 느려지지 않는다. 미션은 안전설계에 맞서서가 아니라 안전설계에 의해 달성된다.

1. 유지 vs 뒤집기 — (A)데이터 레일 [법적·학습무관] vs (B)역량 제한 [선택]

KEEP — 법적 의무, DATA에 관한 것, 학습 안 건드림

  • K1 만 14세 미만 가입 차단 — PIPA §22-2 / COPPA / GDPR Art.8
  • K2 위기→청소년 핫라인(1388), 항상 ON·tier무관 — 진짜 하드레일
  • K3 미성년 egress 잠금(ads/share/analytics/training=false) — Children's Code Std7 / GDPR Art.25
  • K5 불변 동의 원장 — GDPR Art.7(1)
  • K6 민감(정신건강) 별도 동의 — GDPR Art.9 / PIPA §23
  • K10 비임상 어휘 가드 — 이건 enabling(자유롭게 가르치게 함)
  • K11 AI-vs-인간 고지 — 신설 필요(EU AI Act Art.50 / 韓 AI기본법 §13)
  • K12 아동 DPIA — 최우선 부채(미작성)

FLIP — 법이 아닌 선택 → enablement로 교체

  • F0 "미성년 AI 기본 OFF" → 돌아가는 코드엔 없음. 명칭만 "data-protection rails"로 개명, AI는 풀 ON 유지
  • F1 recommendations 미성년 OFF → ON + "왜 이걸?" 투명성 + 원터치 OFF(egress 없음 = 학습용)
  • F2 scaffold가 유료 Advisor에만 → tier무관 무료 경로에 부착(가격에 의한 (B)캡 방지)
  • F3 long_term_memory opt-in → retention(A,잠금) / personalization(B,graded) 분리
  • F4 비임상 가드 = 미성년 제한? 아님. K10 유지

2. 학습자 면허 모델 — OFF/ON의 대체 (보조바퀴만 사라짐)

단계AI 역량보이는 scaffolding과의존 friction
L0 학습자
(신규 기본)		Fullprovenance 1줄 + evidence 칩 + verify-nudge + 리터러시 micro-lesson 회전5연속 무수정 수용 시 "이번 건 반박해볼래?"

역량 축 = 모든 미성년에게 MAX·상수(코드가 이미 그러함). competence 신호(evidence_open_rate·ai_override_rate·flagged_wrong)는 전부 first-party·on-device(미성년 external_analytics 잠김 → 외부 송출 0). 과의존 회귀 시 가이던스가 두꺼워질 뿐 AI는 약해지지 않음 — 처벌 아닌 harm-reduction. 성인도 opt-in 가능 → "kid mode" 아니라 "좋은 AI 시민성"(non-stigmatizing). 비대칭 hysteresis(승급 ≥0.6·14일 / 강등 <0.4·7일)로 깜빡임 방지.

3. 추가할 ENABLING 역할 — "제한 말고 다른 것"의 답

★ Lane 1 · AI-리터러시 / Scaffolded-Autonomy (센터피스 = 미션 그 자체)

프롬프트 리터러시·출력 비판평가·메타인지(UNESCO/DigComp). users.learner_stage 1열 + 후속 카드에 provenance/evidence/verify-nudge/micro-lesson/"이거 이상한데" 플래그. 기존 XP 엔진 재사용 → 아트 비용 ~0.
Claude(교육로직) · Codex(렌더+copy) · Gemini-AG(14세 a11y QA) · Grok(UNESCO 1차출처) · 와이어 §29→§26→aha-moment · §30 paywall 금지 · §35 ai-debate 필수 · leverage 최상/effort 중

Lane 2 · "Show-your-work" 투명성을 모든 AI 표면으로

자동화 편향·오라클화 방지. 챗에 이미 있는 grounding strip + 레퍼런스 drawer를 Advisor/persona 카드로 일반화(create.ts가 이미 evidence[]+DOI 반환). Codex · Gemini-AG · leverage 높음 / effort 낮음(절반 구현됨)

Lane 3 · Calibrated-humility "금지 프레이밍" copy-law (CI 강제)

sycophancy·자기상 왜곡 방지. 어휘가드 확장으로 "we know you better than yourself"류 차단 + dual display 의무("너는 X라 했고 기록은 Y에 가까워 — 둘 다 사실일 수 있어"). Claude(정본) · Grok · leverage 높음 / effort 낮음

Lane 4 · "내 생각 먼저" 인지강제 (skippable·fade)

과의존 직격(Buçinca 2021: 먼저 견해 commit→overreliance↓). 응답 공개 전 한 줄 해석→나란히 비교. friction caveat 내장(XP 보상+skippable). Codex · Gemini-AG(14 vs 17세 QA) · §35 토론(default-on 논쟁)

Lane 5 · "AI는 네 치료사가 아냐" 경계 온보딩 + Art.50 고지 (1surface)

"false sense of competence"를 이름으로 가르침. 온보딩 1스크린(나는 AI다·틀릴 수 있다·확인법·결정은 너·어떤 건 사람에게). K11 법적 고지 동시 충족. Codex · Claude(법문구) · Grok(KR/글로벌 변형) · leverage 높음(1타2피)/effort 낮음

Lane 6 · 과의존 self-mirror / 의존 코치 (캡 아님)

파라소셜 의존 방지(APA 2025). cadence 급증 시 코칭 한 줄("한 사람에게 한 문장이라도 말해볼까?"). 명시적으로 분/일 캡 아님 — 진짜 도움받는 아이를 안 벌함. Gemini-AG · Codex · Grok · leverage 중상/effort 중

Lane 7·8 · "Check it yourself" 검증 카드 + recommendations ON + 데이터주권 뷰

evidence payload를 인터랙티브로(동의·의심·검증). F1 실행(미성년 recommendations ON+투명성). "내 계정에서 나간 것 보기" → 데이터 주권 리터러시. Codex · Gemini-AG(egress 경로 audit) · §35 토론

4. 진짜 하드레일 — 단 5개, 전부 데이터/안전/투명, AI 교육 비제한

가르침으로 대체 불가 + 법적 의무. "enablement" 명목으로도 약화 금지:
  1. 위기→인간 핸드오프 — 결정론·인간작성 고정 템플릿(LLM 생성 절대 X)·미성년 1388·3중 차단. AI가 멈추고 사람에게 넘김.
  2. 동의 전 민감정보 egress 없음
  3. 미성년 데이터로 training/ads 없음
  4. 알려진 isMinor 무단 다운그레이드 없음
  5. AI-vs-인간 고지(anti-deception, Art.50 — 현 빌드 갭, 신설)
과의존은 코칭(분/일 캡 아님). CSM/Stanford "18세 미만 companion 금지"(2025.11)는 judo — 2nd-B = scaffolded reflection + 하드 위기 핸드오프 + 리터러시 커리큘럼이지 companion bot 아님을 §35 토론에서 입증(D-19).
