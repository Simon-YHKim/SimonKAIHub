SimonKstack Criteria Applied

Designed States

Loading, error, empty, and disabled states must all be explicitly designed. Empty cannot double as error.

Anti-slop

Weak error-handling and UI defaults are slop classes. This is not cosmetic: it changes what users believe happened.

Required Fix

  • Add route-level load state to Research, Trinity, and Insights.
  • On query error, preserve existing data when possible and show persistent localized recovery.
  • Keep native alerts for foreground user actions, not background route-load failures.
  • Prove both true empty and forced load-error states in KO/EN.
  • Add a regression guard so load error cannot render the true empty component.
