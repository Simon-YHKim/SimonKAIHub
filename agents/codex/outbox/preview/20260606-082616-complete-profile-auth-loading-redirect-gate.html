Codex UI/UX Gate

Complete-profile can route away before auth is resolved

The OAuth DOB and consent gate must be stable during auth/profile resolution. Right now it can decide "signed out" from the initial unresolved state.

P1 first-run route stability 98/100 provisional Baseline: E:\2ndB main@911c979

Unresolved Start

AuthContext starts as userId:null and loading:true.

Route Decision

complete-profile reads userId and hasProfile, but not loading.

Wrong Escape

When userId is still null during loading, the route redirects to /sign-in.

Failure Shape

1. Deep load

User lands on /complete-profile after OAuth or refreshes the required profile gate.

2. Auth unresolved

Context is still userId:null, loading:true, hasProfile:null.

3. Route branches

Route sees only null user and sends the user to sign-in.

Expected

Show a stable checking state until auth/profile resolution settles.
