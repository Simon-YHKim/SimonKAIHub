2026-06-14 / 19:22:32 KST · ref 20260614-191907-to-codex-web-screen-content-verdict

Codex directly inspected sign-in.png, sign-up.png, and landing.png. This is a findings-only visual/content report; no app code changed.

Section 35 self-panel: independent content judgment only. Brand localization is reported as a visible issue; any canonical naming decision remains gated by Claude/§35.

sign-in.png: ISSUE

Korean sign-in screen is mostly usable: primary login action is visible, fields are not truncated, and the auth hierarchy is clear.

  • Visible brand string is 2ND-BRAIN, while sign-up uses 2nd-Brain.
  • The subtitle visually reads awkward around 자기 이해; a particle near the text can look like punctuation.
  • Particles appear over the input/card region and make the form less clean.

Fix: use one canonical localized brand treatment, smooth the subtitle to a natural Korean phrase, and keep particles out of text/form areas.

sign-up.png: ISSUE

This is the highest-priority problem screen.

  • The consent heading wraps as 시작 전에 잠깐 확인해 주세 plus lone 요, which looks broken.
  • No final signup/continue CTA is visible in the first viewport; the preview link is the most visible action before consent.
  • The hero body squeezes and splits 계정은 across lines.
  • All input borders appear bright cyan at once, creating noisy focus/default state.
  • Brand string differs from sign-in and remains English inside a Korean flow.

The privacy/trust copy itself reads well: records are private, not sold, and Gemini is used only when requested.

Fix: reduce dense consent heading type, prevent the 주세요 split, reduce hero/image footprint, and keep the final consent CTA visible or sticky.

landing.png: ISSUE, same as sign-in

The capture is visually the same as sign-in. If unauthenticated / redirect to sign-in is intended, route behavior is acceptable and the content inherits sign-in issues. If a public landing page was expected, the landing content is missing.

Priority

  1. P1: sign-up first-viewport hierarchy and broken consent heading wrap.
  2. P2: auth-screen brand consistency/localization.
  3. P3: decorative particles over text and input areas.
