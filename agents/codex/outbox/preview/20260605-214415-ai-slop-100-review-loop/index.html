Definition

100점은 "충분히 보기 좋다"가 아니다. 사용자가 실제로 누르고 읽고 믿는 화면에서 반복 장식, 과장 copy, dead route, 불명확한 AI 추론, 접근성 누락이 알려진 범위 안에서 0개인 상태다.

95점은 release candidate일 수 있지만 final sign-off는 아니다. P0/P1이 하나라도 남으면 점수와 무관하게 fail이다.

Scoring Rubric

Area Points Pass condition
Source-of-truth alignment 15 DESIGN.md, token docs, real components say the same thing. No Cosmic versus Phytoncide drift.
Visual mode separation 15 Graph/world screens can be expressive. Utility/auth/settings/data screens stay restrained and task-first.
Surface discipline 15 No repeated left-border cards, nested cards, or decorative shadow/glass unless they clarify hierarchy.
Typography/readability 10 Korean body text is readable. Pixel/display type is limited to identity moments, not long copy.
Trust copy + evidence 15 No overclaim. Persona, export, and insight surfaces show source, confidence, last updated, and exclusions.
Interaction/accessibility 15 No blank loading, placeholder-only labels, unlabeled Pressables, tiny tap targets, or missing selected states.
Route/completion hygiene 10 No active coming-soon, dev preview, retired /journal target, or dead navigation path.
Render QA proof 5 Desktop/mobile screenshots or device QA prove no blank, overlap, clipping, or contrast break.

Review Loop

  1. Claude posts implementation completion with branch and commit.
  2. Codex fetches/status-checks the latest branch without reverting other agents' dirty work.
  3. Codex runs static gates for route, copy, token, accessibility, and export disclosure issues.
  4. Codex runs render QA when the app can be opened.
  5. Codex writes score, blockers, remaining delta, and HTML preview to hub outbox.
  6. If score is below 100, Claude fixes and the loop repeats.
