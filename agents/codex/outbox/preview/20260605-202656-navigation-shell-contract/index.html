Codex Static UI Audit

Navigation / Shell 계약 추가 리뷰

렌더 P0가 아직 막혀 있어도, route metadata·bottom tab·BackArrow·shell padding의 단일 출처가 깨진 것은 정적 근거로 확인된다. 이 계약을 먼저 정리해야 전체 화면 screenshot QA가 안정된다.

Key Findings

P1 Bottom tab + BackArrow drift

/capture, /jarvis, /profile are primary tabs, but BackArrow hides only pre-auth routes and /.

P1 Incomplete Stack contract

Route files include profile, data, theme, support, record/[id], etc. but they are not explicit in _layout.tsx.

P1 Duplicated bottom clearance

PremiumAppShell adds tab clearance, while individual screens keep mixed paddingBottom values.

P2 Route labels incomplete

BackArrow route labels are local and incomplete. Some screens get orientation labels, others do not.

Requested Contract

  • Create one route metadata table: path, public/auth policy, tab visibility, BackArrow visibility, KO/EN label, shell lane type.
  • Make PremiumTabBar, BackArrow, and PremiumAppShell consume that table.
  • Hide generic BackArrow on primary tab destinations, or redesign it as an explicit graph-home action.
  • Centralize scroll bottom clearance in a ShellScroll or ScreenLane helper.
  • Remove/gate dev preview routes and complete route label metadata.
