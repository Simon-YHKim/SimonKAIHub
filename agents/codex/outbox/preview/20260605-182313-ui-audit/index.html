2nd-B 전체 UI 재검사

정적 검사와 실제 웹 렌더 캡처를 함께 봤다. 브랜드 방향은 강하지만, 모바일 반응형 폭 제어와 DESIGN.md 규칙 실행 사이에 launch-blocker급 간극이 있다.

검사일: 2026-06-05 KST 대상: E:\2ndB 라우트: src/app 전체 43개 브라우저: Chrome headless

결론

P0

기본 로컬 환경에서 앱이 첫 렌더 전에 죽음

현재 `.env`의 `EXPO_PUBLIC_SUPABASE_ANON_KEY`가 비어 있지는 않지만 20자 미만이라 zod 검증에서 즉시 crash가 난다. UI 감사도 dummy 공개값을 별도 프로세스에 주입해야 진행 가능했다.

P0

모바일 가로 오버플로

`/sign-in`, `/sign-up`, `/manual`, 그래프 프리뷰에서 390px 폭 기준 오른쪽 콘텐츠가 잘린다. 첫 인상 화면과 약관 화면이 동시에 깨져 launch blocker로 본다.

P1

DESIGN.md와 구현의 시각 규칙 불일치

no gradient, no glass, no dark shadow 규칙과 다르게 app shell, panels, buttons, graph labels가 gradient, glass copy, shadow props를 구조적으로 쓴다.

P1

정보구조 잔여 부채

`/journal`, `/imagine`, `/mbti`, `/trinity`, dev preview routes가 아직 사용자 경험 또는 route tree에 남아 있다. "담기 단일 입력"과 검증된 자기이해 포지션이 흔들린다.

주요 발견

P0

Auth screens width policy 없음

  • 데스크톱에서는 로그인 폼이 화면 폭 대부분을 차지해 집중도가 떨어진다.
  • 모바일에서는 폼과 hero 이미지가 viewport 오른쪽을 넘어간다.
  • 권장: auth form에 `width: "100%"`, `maxWidth`, `alignSelf: "center"`를 일관 적용하고 web root overflow를 QA한다.
P1

긴 본문이 pixel font로 과밀하게 보임

  • `typography.ts` 주석은 긴 Korean copy에 readable sans를 쓰겠다고 하지만 `fontFamilies.sans`는 NeoDunggeunmo다.
  • 회원가입 동의문과 매뉴얼 카드에서 줄바꿈이 조밀하고 clipping이 두드러진다.
  • 권장: heading/labels는 pixel, 긴 문단과 legal copy는 readable alias로 분리한다.
P1

그래프 모바일 fit 부족

  • 390px 폭에서 오른쪽 core label과 top segmented control이 잘린다.
  • 실제 그래프가 pan 가능한 의도여도 첫 화면에서 node label이 잘리는 것은 완성도가 낮아 보인다.
  • 권장: initial scale/center를 viewport 기반으로 계산하고 dev controls는 production에서 제거한다.
P2

Expo web warning noise

  • `shadow*` deprecated, `pointerEvents` deprecated, `useNativeDriver` unsupported warnings가 반복된다.
  • 시각 결함과 직접 동일하지는 않지만 web polish와 성능 신뢰를 떨어뜨린다.
