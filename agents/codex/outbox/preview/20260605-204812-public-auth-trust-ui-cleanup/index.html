P1 Findings

P1 Public Support & Privacy

/support and /privacy redirect logged-out users. That blocks the exact users who need trust evidence before creating an account.

  • Make Support public.
  • Make Privacy public policy + signed-in controls, or split /privacy-settings.
  • Expose Privacy, Support, Permissions, Manual from auth footers.

P1 Auth Trust Strip

Sign-in/sign-up only link to Manual. Sign-up asks for consent and DOB without adjacent export/delete/no-diagnosis context.

  • Add compact trust strip: Privacy, Support, Permissions, Export/delete.
  • Add four bullets near consent: data yours, export before delete, source-backed, no diagnosis.
  • Keep touch targets at least 44px.

P1 Auth Error UX

Auth errors and password reset are system alerts. Password reset says coming soon, which reads unfinished at the most sensitive moment.

  • Use inline persistent banners.
  • Replace coming-soon reset with real reset or support mailto.
  • OAuth failure needs retry + support path.

P1 Shared Auth Shell

Sign-in uses a custom cosmic shell; sign-up and complete-profile use PremiumAppShell. The funnel does not feel like one product flow.

  • Standardize AuthShell with lane, hero, status, footer, keyboard-safe scroll.
  • Use one visual grammar across auth screens.
  • Remove route-local bottom padding assumptions.

P2 Findings

Area Issue Requested Direction
/manual / /permissions Useful trust content exists but does not form a strong public cluster. Cross-link Manual, Permissions, Privacy, Support, Sign up, Sign in; mark auth-gated Research as sign-in required or hide.
+not-found Common destinations include /journal, /audit, /persona. Session-aware destinations: public routes when logged out, capture/wiki/settings when logged in.
/onboarding Trust step is behind auth and final route still targets /journal. Route final first-run to Capture or graph root, and expose key trust copy in public auth/manual surfaces.
/oauth-callback Failure path is minimal and lacks support context. Add verifying/failed/retry/support callback panel without raw error leakage.

Implementation Order

  1. Make /support and /privacy readable before login.
  2. Create shared AuthShell and inline status banner.
  3. Add sign-in/sign-up trust strip and sign-up consent trust bullets.
  4. Replace password-reset coming soon and alert-only errors.
  5. Build a public trust cluster across Manual, Permissions, Privacy, Support.
  6. Clean 404 destinations and onboarding /journal route.
  7. Improve OAuth callback transition state.

Release Acceptance

  • Logged-out users can open /support, /privacy, /permissions, and /manual.
  • Sign-in and sign-up footers expose Support, Privacy, Permissions, and Manual.
  • Sign-up shows data ownership, export/delete, source-backed, no-diagnosis trust bullets near consent.
  • No user-facing password reset "coming soon" alert remains.
  • Auth failure states are inline and screenshot-visible.
  • Logged-out 404 users are not sent to retired or auth-gated routes without context.
  • Public/trust routes do not promote /journal as a current destination.
