Inbox Wiki Promotion Identity Gate

Inbox can promote a source row into a wiki page and knows the generated slug, but the completed row's View in wiki action opens only the generic wiki list.

98/100provisional anti-slop score
P2workflow trust gate
/inboxsource promotion flow
lost sluggeneric /wiki hand-off

Finding

/inbox confirms a specific generated page such as [[some-slug]], then shows View in wiki. That action links to /wiki without passing slug, page id, or source id, so users land in the full wiki list and must rediscover the page manually.

Requested Contract

Option A - Exact Page Navigation

Create /wiki/[slug] or a slug param on /wiki that expands/highlights the generated page.

Option B - Query/Highlight Hand-Off

Pass a slug/source param to /wiki, prefill search, and highlight the row while handling deleted pages.

Option C - Honest Generic Shortcut

Rename the action to Open wiki list and keep the generated slug visible or copyable in row state.

Acceptance

  • /inbox -> Generate wiki page -> View in wiki opens or highlights the exact generated page, or the action is clearly generic.
  • The identity hand-off works after reload, not only from immediate in-memory state.
  • Deleted or missing generated pages have a clear no-match state.
  • KO and EN copy match the chosen behavior.
  • Tests or route-level checks cover the source row to wiki page transition.

Re-Review Path

/inbox source row -> Generate wiki page -> success -> View in wiki -> exact generated wiki page or truthful generic list
