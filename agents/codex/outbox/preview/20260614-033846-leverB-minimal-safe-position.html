Lever B is feasible only as device-local pending

2026-06-14 / 03:38:46 KST · ref leverB-debate-20260614

Conditional yes: local pending first, Records only after profile and consent.

Safe Shape

  • Pre-account captures one plain text note into a separate device-local queue.
  • No LLM, Supabase, Storage, graph, wiki, OCR, clipper, or Records claim before DOB and consent.
  • Local deterministic crisis hinting may run, but post-profile import must re-run createRecord().
  • Import is explicit after userId + hasProfile=true, never silent.

Allowed

  • capture.preauthPending.v1 local queue.
  • Plain text only.
  • "Temporary on this device" copy.
  • Retry, delete, export if import fails.

Disallowed

  • Pre-account createRecord().
  • Pre-account captureFromMarkdown() or Storage.
  • Advisor, clipper, OCR, AI classification.
  • "Vault", "synced", "backed up", or "Records" before import.

Reason

  • The current app intentionally gates profileless sessions away from all feature screens.
  • Current record/source writes require userId and rely on post-profile C10/C3/C9 safety paths.
  • AsyncStorage/localStorage are durability tools, not an encrypted privacy guarantee.

Recommendation

Approve only as a scoped activation experiment after DECISIONS captures the product/security boundary.
