Profile Probe Error State Gate

A completed account should not be sent to the profile-completion form because the profile check timed out or failed.

Project: 2nd-B Head: 911c979 Score: 98/100 provisional Priority: P1/P2

Current Conflation

Probe error or timeout

fetchProfile error and timeout fallback both return hasProfile: false.

->

Wrong user-facing state

Root routing interprets false as "profile missing" and redirects to /complete-profile.

Evidence

src/lib/auth/AuthContext.tsx:47-48 profile probe failed -> { hasProfile: false, isMinor: null } src/lib/auth/AuthContext.tsx:58-78 timeout/catch -> fallback src/lib/auth/AuthContext.tsx:133-134 fallback -> { hasProfile: false, isMinor: null } src/app/_layout.tsx:165-167 hasProfile === false -> Redirect /complete-profile

Expected Contract

No user

Signed-out route state. Show sign-in or public route.

Profile present

Allow app entry with confirmed DOB/minor state.

Profile missing

Only confirmed absent row routes to completion.

Profile unknown

Error or timeout shows retryable profile-check state, not completion form.

Acceptance

  • Introduce explicit profile status such as checking, present, missing, and error.
  • Redirect to /complete-profile only for confirmed missing profile.
  • On query error or timeout, render a branded retry path with sign-out/support fallback.
  • Keep AI/crisis feature screens blocked while profile status is checking or error.
  • Add tests for present, missing, query error, and timeout cases.
