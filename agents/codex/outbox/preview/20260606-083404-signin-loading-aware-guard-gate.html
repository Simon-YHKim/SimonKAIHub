Codex UI/UX Gate

Sign-in can show the wrong workflow while auth resolves

The route redirects signed-in users only after userId is populated. During unresolved auth, it renders the login form.

P2 auth route-state 98/100 provisional Baseline: E:\2ndB main@911c979

Failure Shape

1. Session exists

User is signed in or returning from a prior tab session.

2. Auth unresolved

AuthContext is still userId:null, loading:true.

3. Sign-in renders

The form appears because the route only guards on populated userId.

Expected

Show checking state, then route to the correct destination.

Code Evidence

  • sign-in.tsx:48 reads only userId from auth context.
  • sign-in.tsx:60-64 redirects signed-in users only after userId is truthy.
  • AuthContext.tsx:27-31 and :87-91 start unresolved with loading:true.
  • AuthContext.tsx:131-140 marks first signed-in profile resolution as loading.
  • _layout.tsx:169-171 can return children directly after intro has played, so route-local loading guards still matter.
const { userId } = useAuth();

if (userId) return <Redirect href="/" />;

Required Contract

  • /sign-in waits for auth loading before rendering a form.
  • Signed-in complete-profile users route to /.
  • Signed-in missing-profile users route to /complete-profile.
  • Signed-out users see the normal sign-in form.
  • /sign-in, /sign-up, and /complete-profile share the same route-state primitive or documented contract.
