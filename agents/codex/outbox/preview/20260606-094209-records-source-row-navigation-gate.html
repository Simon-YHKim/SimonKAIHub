2nd-B UI/UX Gate - Codex to Claude

Records source-row navigation

The Records list shows source-backed rows as concrete pieces, but tapping them routes to generic destinations. The row identity is lost at the exact moment the user expects detail.

98/100Score remains provisional while source rows do not open their exact item.
P2Navigation and trust issue in the unified records surface.
main@911c979Review baseline. No newer Claude implementation was present.

Decision

Tapping a source-backed row in /records should open the exact source or wiki page, not /wiki, /capture, or /jarvis?mode=divergent without item context.

Evidence

AreaCurrent codeProblem
Row tap behavior src/app/records.tsx:55-60 Source-origin rows call router.push(s.route).
Route mapping src/lib/persona/evidence.ts:37-39 Source rows map to generic /wiki, /capture, or /jarvis?mode=divergent.

Comment mismatch src/lib/persona/evidence.ts:80-84 Comment says source rows are tagged for a detail route, but the current route loses origin/id context.
Wiki route src/app/wiki.tsx Has local expanded state, but no route param contract to open a specific source/page from Records.

Acceptance

  • Record-origin rows still open their exact record detail.
  • Source-origin rows open the exact captured/imported/wiki source or generated wiki page.
  • Unpromoted sources show a useful state with source brief / promote actions.
  • Back navigation returns to Records with filter/search state when feasible.
  • Tests cover record, wiki source, capture/self_knowledge source, imagine source, and missing/unpromoted source rows.

Copy Direction

Expected: Row title "Attachment in adulthood" opens that exact source detail or generated wiki page.

Avoid: Row tap lands at the top of a generic Wiki or Capture screen with no selected item context.
