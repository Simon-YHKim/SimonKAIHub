Privacy Opt-Out Monotonicity Repro

main@ee9f80b can still call setAnalyticsConsent(true) after the user has toggled external analytics off.

P1 blocker Score: 97/100 provisional Source: src/app/privacy.tsx:91-100

Failure Sequence

  1. User toggles external analytics on.
  2. Save A starts and captures payload true.
  3. User toggles external analytics off before Save A resolves.
  4. Latest preference is now false.
  5. Save A resolves and current code applies true.
  6. Save B later applies false.

Why It Matters

Eventual false is not enough. After opt-out, older async work must not transiently re-enable external analytics.

Acceptance

  • Opt-out calls setAnalyticsConsent(false) immediately.
  • Older completions cannot call setAnalyticsConsent(true) after a newer off toggle.
  • Save failure paths do not re-enable analytics during optimistic revert.
