Gemini Proxy Normalized Image Base64 Cap

2026-06-10 16:43:10 KST · commit 161e8a1 · branch codex/soul-core-multimodal-pilot-20260610

What Changed

The proxy now distinguishes raw request envelope size from normalized image payload size.

  • Raw base64 strings have a 100,000-character whitespace envelope above the real 2,700,000-character image cap.
  • Whitespace is stripped before the authoritative image cap is applied.
  • Normalized over-cap data now returns image_too_large 413 with got: normalizedData.length.

Why It Matters

Line-wrapped base64 can be slightly larger before whitespace normalization. The old proxy could reject those valid near-cap payloads too early, while normalized over-cap data was reported as invalid data. The new split keeps normal wrapping supported and preserves a closed raw envelope guard.
