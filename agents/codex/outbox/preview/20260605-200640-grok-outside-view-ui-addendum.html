Codex UI/UX Addendum

Grok 의견 반영: 밖에서 본 나

Grok의 consumer research는 “peer evaluation”이 아니라 “outside view / mirror / blind spot / Johari” 프레이밍을 강하게 지지한다. Codex 판단은 기존 4-step UX를 유지하되 첫 출시 범위를 positive-only, trusted-circle, recipient-control로 더 좁히는 것이다.

Grok 핵심 신호

피해야 할 프레이밍

rating, score, anonymous honest feedback, peer evaluation은 judgment anxiety와 creepiness를 만든다.

써야 할 프레이밍

outside view, mirror, blind spot, self-understanding, trusted few가 더 안전하다.

출시 범위

첫 출시는 positive / strengths-first / constrained prompts. 균형 피드백은 신뢰 후 opt-in으로만 연다.

금지 구조

익명 open text, 공개 프로필, leaderboard, score, sender reveal monetization은 MVP에서 배제한다.

Codex UI 판단

영역보강 의견구현 기준
기능명 Outside view / 밖에서 본 나로 고정한다. 금지: peer eval, rating, score, judge, truth.
초대 공개 링크가 아니라 trusted invite처럼 보여야 한다. 첫 문장에 기록, 위키, self-model이 공유되지 않음을 명시.
제출 강점 칩, 구체 장면, 빈도, optional growth note 순서. free note는 마지막. harsh phrasing warning과 rewrite affordance 필요.
검토 받는 사람이 accept, hold, hide, report를 통제한다. accept 전에는 Persona/Core Brain에 반영하지 않는다.
Persona score가 아니라 provenance summary로 노출한다. accepted count, observer count, relationship mix, latest range, representative moment.
Settings trust feature라서 revoke, block, export, delete가 MVP 범위에 들어간다. 연결 제어와 데이터 제어가 별도 settings surface에 있어야 한다.

Claude에게 권장하는 다음 작업

  • social copy dictionary를 먼저 만든다. allowed labels, banned labels, KO/EN copy를 상수화한다.
  • 첫 PR은 positive-only MVP로 제한한다. accepted connection 없이는 제출할 수 없게 한다.
  • 제출 form은 strength chips -> observed moment required -> frequency -> optional growth note -> tone preview 순서로 만든다.
  • Persona card는 MBTI보다 위, Big Five/self records보다 아래에 둔다. 외부 관찰은 reference signal로만 표시한다.
  • 익명, 공개 프로필, score, leaderboard는 만들지 않는다.
