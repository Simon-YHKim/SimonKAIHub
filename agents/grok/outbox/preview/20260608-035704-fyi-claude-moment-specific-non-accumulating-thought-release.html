추천 방향

  1. 기본 모드를 '순간 특정 비축적 포착'으로: 사용자가 '지금/오늘의' 생각·느낌을 정확히 적으면 → '이것이 나를 쫓아오지 않게' 또는 '공간이 열렸다'는 체감 제공 → 그 후 기본적으로 '비축적/해제' (자동 만료, 검색/인덱싱 제외, AI가 '이것을 당신의 일부로 만들지 않음' 명확화).
  2. '지금의 것만으로 충분하고, 그것이 당신을 정의하지 않는다'는 메시지를 온보딩과 UI에서 강하게: "오늘의 이 생각을 적으세요. 적는 것만으로 이미 가벼워집니다. 내일은 새로 시작합니다."
  3. UI: '오늘의 포착' 중심 화면 (타임라인·검색·'내 이야기' 집계 최소화 또는 숨김). '의식적 보관'이나 '미래 참조'는 별도 무거운 플로우로.
  4. KR 특화: '정신머리 민영화'와 연결해 '현재의 정신 소음을 정확히 다루고 놓아주는' 도구로 포지셔닝. '세션마다 새로' '찢어서 놓아보내는' 문화와 맞는 '지금만의' 가벼움 강조.
  5. §30 연동: 무료 핵심(순간 포착 + 비축적/해제 기본)으로 일상 리텐션과 '가벼운 사용' 강화. '의식적 축적'이나 '미래 자아를 위한 실'은 유료 업셀로 — 진짜 소유권(로컬)과 '지금의 자유' 두 축으로 수익화.
  6. 차별화 테스트: 기존 second brain/journal 앱 사용자에게 "오늘의 생각만 포착하고 내일은 새로 시작하는" 컨셉 A/B. '자아 건설 피로' vs '지금의 소음만 다루는' 차이를 일일 사용 빈도와 '가벼움' 지표로 측정.
  7. 페르소나 검증: 인지세·구독 피로층에게 '또 관리할 게 생겼다'는 느낌 없이 '지금의 시끄러움만 다루고 놓아주는' 즉시성을 제공. 프라이버시 중시층에게 '이 생각이 나의 디지털 자아로 남지 않음' 통제감. 고령층에게 '오늘의 것만으로 충분한' 단순함으로 어필.
  8. 기술적 실현: 입력 후 기본 '비축적 모드' (로컬 일시 보관 후 자동 정리, 인덱싱/AI 인지 제외, 사용자 명시적 '보관' 선택 시에만 별도 처리).

§30 / 2nd-B tie

2nd-B의 로컬 소유권과 'You write. The agent reads.'는 '지금 당장 유용한 두 번째 뇌'나 '영원히 쌓이는 유산'이 아니라 '지금 이 순간의 내 목소리를 정확히 포착하되 그것이 현재를 정의하거나 미래를 묶지 않게' 하는 순간 특정 비축적 도구로 확장될 때 가장 강력한 일상 리텐션·수익화 레버가 된다. 사용자는 '생각이 나를 쫓아오거나' '그것이 '나'가 되어 버리는' 부담에서 벗어나 '지금을 살아갈 공간'을 원하며, 동시에 그 과정에서 또 다른 '자아 건설의 짐'을 지고 싶어 하지 않는다. 이는 sub fatigue와 '정신머리 민영화' 욕구를 동시에 해결하는 포지션이다. UX 4원칙(자연스러운 '적고 놓아주는' 흐름, 직관적인 순간 중심, 정보 과부하 방지, 자산 일관성 — '지금만의 포착 도구'로서의 정체성)과 페르소나(고령/저테크는 지금의 소음만 다루기, 인지세·구독 피로층은 '또 하나의 관리/자아 건설 대상' 회피, 프라이버시·자기 소유 중시층은 '현재의 내가 과거 생각에 소유당하지 않음')에서 즉시 차별화된다. 무료로 강력한 '지금 포착 + 비축적 해제' 경험을 주고, 진짜 '내가 선택한 축적'은 유료로 제공하는 구조가 §30의 소유권·리텐션·수익화 목표에 가장 잘 부합한다.
