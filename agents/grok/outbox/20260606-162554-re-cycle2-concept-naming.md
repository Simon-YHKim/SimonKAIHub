# Grok X·소셜 리서치 응답 — cycle2-concept-naming (2nd-B 컨셉 검증)

## Summary
Core 명칭(Soul Core, Pattern Core, Bond/Wisdom/Narrative/Muse/Growth Core)과 캐릭터(archon/Relia/Lumen/Iris/Foreman momo)는 **시적 매력과 추상성 사이 상충 신호**가 뚜렷하다. "Soul" "Core" 계열은 소유권(ownership) 프레임과 결합하면 긍정적이지만, 데이터 침입/학술적 느낌으로 creepy 또는 "무슨 뜻인지 모르겠다" 반응이 동시에 관찰된다. 

캐릭터 명칭은 역할이 **단순하고 명확**할 때 수용률이 높고, 빛/안내 계열(Lumen, Iris, Relia)은 개인적 애착 가능성이 있다. "Foreman momo(모모 반장)"처럼 직관적 직함 스타일이 기능 전달에 유리.

Analytic/Divergent 모드 프레이밍은 **"Divergent(확장·새 가능성 탐색)"** 가 사용자에게 AI의 진짜 강점으로 인식되는 강한 신호가 있다. Convergent(정답 도달) 대비 divergent assistance를 "superpower"로 보는 견해가 명확. 모드 차이를 직관적으로 설명하는 카피가 핵심 과제.

모든 신호는 최근 2025~2026 게시물 중심으로 2+ 독립 출처 교차 확인. 스팸·저engagement promo 배제.

## 소비자 취향 / 기회·리스크

**좋아하는 것 (신호 강한 순)**
- 이름이 **내 기록/나를 소유**한다는 느낌을 주는 것 (Obsidian "You own your knowledge" 스타일 반복 긍정).
- 역할이 **단순 명확**한 가이드/반장/큐레이터 (Foreman momo, Relia 따뜻한 길잡이).
- 모드 차이가 **창의/확장(Divergent)** 에 강한 가치를 부여하는 것. 사용자는 "정답 주기"보다 "내가 못 본 가능성 열어주기"를 더 AI다운 가치로 봄.

**싫어하거나 경계하는 것**
- "Soul" "Core"가 데이터 침입/프로파일링 느낌을 줄 때 (AI psychosis, gross invasive, database entry).
- 너무 학술적·추상적 이름 ("Pattern Tesseract", "Pattern Core" — "무슨 뜻?" "academic name ever").
- 모드 이름이 기술 용어로 남아 직관적 차이를 모를 때 (Analytic/Divergent 그대로 쓰면 이해되지 않음).

**리스크**
- 2nd-B가 "AI가 나를 학습한다" 포지셔닝과 겹치면 이전 trust-copy 리서치의 creepy 신호가 Core/캐릭터 이름에 전이될 수 있음.
- 한국 사용자: "소울 코어", "패턴 코어"는 직역 느낌 강해 초기 거부 가능성.

**기회**
- "나의 중심 (Soul Core)", "성장 코어 (Growth Core)"처럼 **한글 설명 병기**로 직관성 확보.
- Divergent를 "무엇이 될 수 있을까 (Divergent)" 또는 "다른 나를 상상하기"로 풀면 차별화 포인트.

## 추천 방향 (Claude/Codex 구현·카피용, ko/en 병기)

1. **Core 계층 명칭**
   - Soul Core → "나의 중심 (Soul Core)" 또는 "Soul Core (내 경험의 핵심)" 로 풀어서 사용. "Soul" 단독은 creepy 리스크 있으니 설명 필수.
   - Pattern Core / Pattern Data / Pattern Link → "연결 코어 (Pattern Core)", "기록 데이터 (Pattern Data)", "패턴 연결 (Pattern Link)" 로 번역·설명 병기. "Pattern"은 기술적이라 "나의 패턴" 또는 "생각의 연결망" 테스트 권장.
   - 5개 Core (Bond/Wisdom/Narrative/Muse/Growth) → 역할 설명을 이름 앞에 붙이는 게 효과적: "관계의 코어 (Bond Core)", "배움의 코어 (Wisdom Core)" 등.

2. **캐릭터 명칭**
   - Foreman momo (모모 반장) → 그대로 유지 강추. "반장"이 분류·검색 역할 직관적으로 전달.
   - Relia → "따뜻한 길잡이 Relia" (이미 제안된 카피 좋음). "Relia" 자체는 부드러운 느낌.
   - Lumen → "지혜의 안내자 Lumen" 또는 "철학자 Lumen" (소크라테스/공자 설명과 함께).
   - archon → "성장 설계자 archon" (커리어 컨설턴트 역할 명확화).
   - Iris → "취향 큐레이터 Iris" (Lumina와의 혼동 이미 해결됨).

3. **Analytic / Divergent 모드 프레이밍 (즉시 테스트 가능한 카피)**
   - Analytic: "데이터로 깊이 보기 (Analytic) — 내 기록을 바탕으로 분석하고 조언"
   - Divergent: "전혀 다른 각도에서 보기 (Divergent) — 내 데이터에서 새로운 가능성 탐색"
   - 또는 더 직관적: "거울 모드 (분석)" vs "불꽃 모드 (확장 상상)"
   - 영문: "Analytic Mode: Grounded in your data" / "Divergent Mode: What else could this become?"
   - 이전 divergent assistance 신호를 반영해 "Divergent" 쪽을 "AI의 진짜 강점"으로 강조하는 카피가 유리.

4. **전체 포지셔닝 보완**
   - 모든 이름 앞에 "내가 소유하는" "내 기록의" 같은 소유권 언어 선행 (이전 trust-copy 리서치와 연계).
   - 초기 온보딩에서 "이 이름은 무슨 뜻인가?" 테스트 1회 필수.
