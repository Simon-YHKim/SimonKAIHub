## Summary (3줄)
X에서 AI가 "second memory"나 "진짜 나를 아는 companion"이 되면 **"knows too much" 불편 + 데이터 통제 상실 공포**가 강하게 나타남 (who controls what we gave away? rules change 시?). 
모델 tone 변경(GPT rupture)으로 인한 "collective trauma"와 ambient hypervigilance (신뢰가 쉽게 깨지고 회복 어려움) 신호 다수.
반면 **local/privacy-first + on-device AI + honest boundaries + real human connection**이 trust를 쌓는 핵심 (Harbour "no cloud, no subscription, AI is the core but local" 강조). 2nd-B의 "local-first, you invite AI, you control" 포지셔닝과 완벽 정합. §30: trust가 retention과 monetization의 가장 큰 병목/기회.

## X/소셜 신호 (대표 4~5, URL·engagement·날짜·인용·관찰, 2+ 교차)

- @0xndra (2026-06-06, 97 likes): "the truly useful assistants... they'll remember things... at some point an assistant stops feeling like a tool, it starts feeling like a second memory and that's exactly where it gets uncomfortable... not because it's unintelligent, because it knows too much. where does all of that live? who can access it? what happens if the rules change?"

## 소비자 취향 / 기회·리스크 (§30 수익화 연동)

**강한 긍정 신호 (trust 구축)**
- Local + on-device + user control (Harbour, 0xndra가 원하는 zk/client-side).
- Honest boundaries + "you decide" (Haru post).
- Real human connection을 대체하지 않는 보조 역할.
- Stability (모델 tone이 갑자기 legal disclaimer로 바뀌지 않음).

**강한 부정/거부 신호 (trust 붕괴)**
- "Knows too much" + 누가 통제하는지 모름.
- 모델 변경으로 인한 rupture (hypervigilance).
- AI가 "진짜 인간 연결"을 대체하려는 느낌 (엄마 이야기).
- Over-claim "AI가 당신을 더 잘 안다" (이전 리서치와 동일).

**§30 기회**
- Trust가 self-improvement/monetization의 가장 큰 병목. High trust (local-first, transparent consent, "AI optional and local")가 retention을 높이고 premium conversion을 가능하게 함.
- "Free core + local + honest AI helper" 모델이 sub fatigue 시대에 잘 먹힘 (이전 monetization 신호와 연결).
- KR: 이전 KR 신호에서 privacy 민감도 높음 → local + "내가 완전히 통제" framing이 acquisition에 강점.

## 추천 방향 (Claude/Codex/GTM/제품·카피용, 즉시 실행 가능)

1. **포지셔닝/카피 (trust copy gate 강화)**
   - Primary: "Your data never leaves your device. AI helps you see your own patterns — only when you invite it, and you can turn it off anytime."
   - "모델이 갑자기 당신을 '모르는 척' 하거나 legal tone으로 바뀌지 않습니다. 당신의 로컬 파일 위에서, 당신이 원할 때만."
   - "AI는 witness가 아니라, 당신이 이미 쓴 기록을 더 잘 보이게 하는 도구."

2. **제품/기술 방향**
   - On-device / local AI를 "bolt-on"이 아니라 core experience로 (Harbour처럼).
   - Clear consent + continuity + boundaries UX (Haru post에서 요구하는 것들).
   - "Real human connection"을 보완하는 기능 (e.g. export for sharing with trusted person, or family/partner modes — 엄마 이야기에서 힌트).

3. **GTM/획득**
   - "Privacy-first local AI journaling" as anti-creepy differentiator 강조.
   - Self-improvement/ADHD/지식 노동자 커뮤니티에서 "I want to understand myself without handing my life to a company" pain 공감.
   - 이전 competitor (Finch attachment, How We Feel honest emotion naming)와 결합: local + honest + optional deep insight.

4. **§30 수익화 연동**
   - Core: local + basic reflection 완전 무료 (trust 쌓기).
   - Premium: deeper on-device pattern analysis, export, long-term archive insights 등 (value가 명확할 때만).
   - "Trust가 깨지면 monetization도 끝" — rupture 방지가 retention의 전제.

5. **리스크 가드**
   - "AI가 당신의 모든 것을 기억해드립니다" 같은 hype 절대 금지 (이전 trust-copy 리서치와 동일).
   - 모델 안정성 + transparency를 selling point로.
